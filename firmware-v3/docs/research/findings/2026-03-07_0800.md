## 🔴 HIGH-VALUE FINDING

### TS3IM — Structural Similarity for Time Series (arXiv:2405.06234)

- **Relevance:** Directly addresses Domain 4 (perceptual similarity for 1D signals) — the exact gap in our metric pipeline where standard 2D image metrics (SSIM, LPIPS) fail on 320×1 LED strip data.
- **Key transferable method:** TS3IM adapts SSIM to time series by decomposing similarity into three dimensions: **trend** (direction of change), **variability** (amplitude dynamics), and **structural integrity** (pattern shape preservation). Claims 1.87× closer to DTW in evaluation results and 50%+ improvement in adversarial recognition vs conventional metrics.
- **How we'd use it:** Extract per-frame LED brightness/colour time series and audio feature time series (energy envelope, spectral centroid, onset strength). Compute TS3IM between the two as a native 1D perceptual similarity score — no need to reshape our 320-pixel strip into a fake 2D image. Can also compare "expected" vs "actual" LED animation trajectories during closed-loop optimisation.
- **Implementation complexity:** Low — the metric is lightweight and operates on 1D vectors. Core implementation is a few hundred lines of NumPy. No GPU required.
- **Layer affected:** L3 (perceptual similarity)

## 🔴 HIGH-VALUE FINDING

### elaTCSF — Temporal Contrast Sensitivity Function for Flicker Detection (arXiv:2503.16759, SIGGRAPH Asia 2024)

- **Relevance:** Provides a perceptually-grounded model of human flicker sensitivity that accounts for luminance, eccentricity, and display area — directly applicable to determining acceptable flicker thresholds for LED strip animations.
- **Key transferable method:** Extended TCSF model incorporating luminance levels, visual eccentricity (centre vs peripheral vision), and display area into a spatial probability summation model. Produces a flicker visibility threshold: below the threshold, temporal changes are imperceptible to humans. Includes the first variable refresh rate flicker detection dataset.
- **How we'd use it:** Replace or augment the current VBench temporal_flickering metric with a perceptually-calibrated threshold. Given our LED strip's physical dimensions, viewing distance, and luminance range, elaTCSF can predict exactly when frame-to-frame brightness changes will produce visible flicker. This lets us set physics-based pass/fail criteria rather than arbitrary thresholds. Also useful for determining minimum effective frame rates at different brightness levels.
- **Implementation complexity:** Medium — requires adapting the model parameters to LED strip geometry and typical viewing conditions. The core math is published; main work is calibration.
- **Layer affected:** L0 (frame quality) / L1 (temporal coherence)

## Finding 3

### CycleSync — Round-Trip Audio-Visual Synchronisation Metric (arXiv:2509.21893, Sep 2025)

- **Relevance:** Novel approach to AV sync measurement that doesn't require paired training data or explicit beat/onset detection — measures whether generated visuals contain enough temporal structure to reconstruct the original audio.
- **Key transferable method:** Feed generated video into a pretrained Video-to-Audio (V2A) model to reconstruct audio, then compare temporal peaks between reconstructed and original audio. Correlation with human judgements r=0.486 (95% CI [0.053, 0.919]) — highest among tested metrics, outperforming AVSyncD.
- **How we'd use it:** Capture LED strip video → run through V2A model → compare reconstructed audio peaks with original audio peaks. If the LED animation encodes enough musical structure that a V2A model can reconstruct the audio's temporal peaks, the visualisation is well-synchronised. This is an elegant indirect measure: good visualisations should be "decodable" back to audio timing.
- **Implementation complexity:** High — requires a pretrained V2A model and video capture pipeline. The V2A model may not generalise well to LED strip video without fine-tuning.
- **Layer affected:** L2 (audio-visual alignment)

## Finding 4

### PEAVS / FAVD — Perceptual Evaluation of Audio-Visual Synchrony (arXiv:2404.07336, Apr 2024)

- **Relevance:** First automated metric specifically designed for audio-visual synchrony "in the wild" with a calibrated 5-point scale grounded in 100+ hours of human annotations across 9 synchronisation error types.
- **Key transferable method:** PEAVS produces a 5-point synchrony quality score. FAVD (Fréchet Audio-Visual Distance) uses covariance matrices from audio-visual features to track AV modality interactions — uniquely responsive to temporal distortions. Pearson correlation with human labels: 0.79 at set level, 0.54 at clip level.
- **How we'd use it:** Score captured LED-video + audio pairs on the PEAVS 5-point scale. FAVD's covariance-based approach could detect temporal drift between audio and LED response that simpler peak-matching misses. The 9 error types (presumed: early/late onset, missing events, phantom events, etc.) provide a taxonomy for classifying failure modes.
- **Implementation complexity:** Medium — model weights and code would need to be available. Main concern: whether the model transfers to 1D LED strip video vs the natural video it was trained on.
- **Layer affected:** L2 (audio-visual alignment)

## Finding 5

### FVMD — Fréchet Video Motion Distance (arXiv:2407.16124, Jul 2024)

- **Relevance:** Addresses motion consistency evaluation through explicit motion features rather than appearance features — complementary to our existing VBench temporal_flickering metric which focuses on frame-to-frame pixel consistency.
- **Key transferable method:** Track keypoints with PIPs++ → compute velocity and acceleration fields per frame → aggregate into statistical histograms → measure Fréchet Distance between distributions. Validated against human perception of motion quality via large-scale human study.
- **How we'd use it:** Track "motion" of colour transitions along the LED strip — peak positions, wavefront velocities, propagation acceleration. Compare velocity/acceleration distributions against reference animations or audio-derived motion expectations. This captures whether motion *feels* right, not just whether pixels are consistent.
- **Implementation complexity:** Medium — PIPs++ keypoint tracking would need adaptation for 1D LED video. The histogram + Fréchet distance computation is straightforward.
- **Layer affected:** L1 (temporal coherence)

## Finding 6

### VMBench — Perception-Aligned Video Motion Metrics (arXiv:2503.10076, Mar 2025)

- **Relevance:** Introduces 5 perception-aligned motion evaluation metrics that achieve 35.3% higher Spearman correlation with human judgements than baseline methods — specifically designed to match human perception of motion quality.
- **Key transferable method:** Five metrics: Object Integrity Score (OIS), Perceptible Amplitude Score (PAS), Temporal Coherence Score (TCS), Motion Smoothness Score (MSS), and Commonsense Adherence Score (CAS). TCS and MSS are directly relevant — they measure whether motion is temporally coherent and smooth from a human perception standpoint.
- **How we'd use it:** Apply TCS to evaluate whether LED animations maintain coherent patterns over time (no jarring jumps). Apply MSS to measure smoothness of colour transitions and pattern movements. PAS could determine whether the visual amplitude of LED response is perceptually appropriate for the audio energy level.
- **Implementation complexity:** Medium — need to access the benchmark implementation and adapt to 1D video.
- **Layer affected:** L1 (temporal coherence) / L3 (perceptual similarity)

## Finding 7

### Beat Tracking as Object Detection (arXiv:2510.14391, Oct 2025)

- **Relevance:** Reframes beat tracking from sequence labelling to object detection, producing confidence-scored beat intervals with multi-scale temporal resolution — directly applicable to improving our audio analysis pipeline.
- **Key transferable method:** Adapts FCOS (Fully Convolutional One-Stage) object detector to 1D audio using WaveBeat's temporal feature extractor + Feature Pyramid Network for multi-scale temporal patterns. Outputs beat intervals with confidence scores, filtered by non-maximum suppression. Multi-scale FPN captures both beat and sub-beat temporal structures.
- **How we'd use it:** Replace or complement current beat detection with confidence-weighted beats. The confidence score per beat enables weighted BeatAlign: high-confidence beats should have tighter LED synchronisation requirements than uncertain beats. The multi-scale FPN output gives sub-beat structure (e.g., 16th note patterns) that could drive finer-grained visual evaluation.
- **Implementation complexity:** Medium — requires adapting the FCOS architecture. Pre-trained weights may be available.
- **Layer affected:** Pipeline (audio analysis)

## Finding 8

### DVAR — Dense Video Artifact Recognition (arXiv:2601.20297, Jan 2026)

- **Relevance:** Fine-grained artifact detection and classification across 10 categories (appearance, motion, camera axes) — could identify specific failure modes in LED visualisation rather than producing a single quality score.
- **Key transferable method:** Dense localisation and classification of generative artifacts, trained on 80k generated videos. Motion-axis artifacts include temporal discontinuities, flickering textures, and unstable motion. Uses Flow-Magnitude-Guided Dynamic Frame Sampling to focus on motion-salient intervals.
- **How we'd use it:** The per-artifact classification could decompose LED quality issues into specific failure types (flicker, discontinuity, motion instability) rather than a single aggregate score. The dynamic frame sampling based on flow magnitude could focus evaluation compute on the most visually active moments (e.g., during beat drops).
- **Implementation complexity:** High — model is trained on AI-generated video, transfer to LED strip video is uncertain. Would likely require fine-tuning.
- **Layer affected:** L0 (frame quality) / L1 (temporal coherence)

## 🔴 HIGH COMBINATORIAL VALUE

### ZeroGrads — Surrogate Gradients for Non-Differentiable Pipelines (arXiv:2308.05739, SIGGRAPH 2024)

- **Source domain:** Computer graphics optimisation
- **The method:** Learns a neural surrogate of an arbitrary black-box objective function *online*, alongside parameter optimisation. The surrogate is trained on actively smoothed samples of the objective, focusing capacity on the local region near current parameters. No pre-training or pre-computed data required.
- **Combinatorial insight:** Our LED rendering pipeline (audio features → rendering parameters → LED frame output → quality score) is **non-differentiable**. We can't backpropagate through the ESP32's rendering code or the camera capture. ZeroGrads learns a differentiable surrogate of the quality-metric-as-a-function-of-parameters landscape, enabling gradient-based optimisation of visualisation parameters without requiring the pipeline itself to be differentiable.
- **What it unlocks:** Composes with ANY quality metric from the primary findings (TS3IM, elaTCSF, BeatAlign, FVMD) — once the metric exists, ZeroGrads provides the gradient-based optimisation loop to minimise/maximise it. This is the missing piece for Domain 7 (closed-loop optimisation).
- **Implementation complexity:** Medium — the surrogate architecture is lightweight. Main challenge is sample efficiency: each evaluation requires rendering + capture + metric computation.
- **Layer affected:** Optimisation (closed-loop strategy)

## 🔴 HIGH COMBINATORIAL VALUE

### Sparse Autoencoders on Audio Latent Spaces — Interpretable Audio Features (arXiv:2510.23802, Oct 2025)

- **Source domain:** Mechanistic interpretability / Audio ML
- **The method:** Trains sparse autoencoders (SAEs) on audio autoencoder latents (EnCodec, WavTokenizer, DiffRhythm-VAE) and learns linear mappings from SAE features to discretised acoustic properties: pitch, amplitude, timbre. Features are monosemantic — each dimension maps to one interpretable audio concept.
- **Combinatorial insight:** Currently our audio-to-visual mapping is hand-coded: spectral bands → colour channels, energy → brightness, etc. SAE-decomposed audio features provide a principled, disentangled audio representation where each dimension has a known semantic meaning. This enables:
  1. **Traceable evaluation:** "LED brightness tracks the SAE amplitude feature with correlation r=0.82" is more interpretable than "corr(|Δ|) = 0.65"
  2. **Per-feature quality metrics:** Separate quality scores for pitch-tracking, amplitude-tracking, timbre-tracking
  3. **Targeted optimisation:** If pitch-tracking is poor but amplitude-tracking is good, optimise only pitch-related parameters
- **What it unlocks:** Composes with TS3IM (compute TS3IM between each SAE feature and its corresponding LED parameter) and with ZeroGrads (optimise per-feature independently).
- **Implementation complexity:** Medium-High — requires running SAE on audio latents in real-time or near-real-time. SAE inference is fast; the bottleneck is the audio encoder.
- **Layer affected:** Pipeline (audio analysis) / L2 (audio-visual alignment)

## 🔴 HIGH COMBINATORIAL VALUE

### VisionReward — Multi-Dimensional Visual Preference Reward Model (arXiv:2412.21059, Jan 2026)

- **Source domain:** RLHF for image/video generation
- **The method:** Decomposes human visual preference into fine-grained, disentangled dimensions. Trains separate reward heads for distinct quality factors, then combines them. Achieves higher correlation with human judgement than single-score reward models because the dimensions are independently calibrated.
- **Combinatorial insight:** Maps directly to our layered metric architecture (L0–L4). Instead of a monolithic "quality score," train a multi-headed reward model where each head corresponds to a metric layer: frame quality, temporal coherence, audio-visual sync, perceptual similarity, aesthetic quality. The disentangled structure means:
  1. Each dimension can be independently validated against human ratings
  2. The optimisation system can target specific weak dimensions
  3. New metric dimensions can be added without retraining the whole model
- **What it unlocks:** Composes with ZeroGrads (multi-objective optimisation using per-dimension gradients) and with the MLLM-as-judge approach (structured rubric → per-dimension scores → trainable reward model that approximates the MLLM at low cost).
- **Implementation complexity:** High — requires collecting human preference data for LED visualisations and training the reward model. But the architecture pattern is immediately applicable for structuring existing metrics.
- **Layer affected:** L4 (aesthetic) / Optimisation (reward signal)

## 🔴 HIGH COMBINATORIAL VALUE

### X-evolve / LLM-Driven Evolutionary Optimisation (arXiv:2508.07932, Aug 2025)

- **Source domain:** LLM-guided program synthesis
- **The method:** LLMs generate "tunable programs" — code with parametric solution spaces. A score-based search algorithm explores the parameter space guided by objective function scores. Requires up to 100× fewer LLM calls than prior methods because the LLM defines the *space* and the search algorithm explores it.
- **Combinatorial insight:** Our visualisation effects ARE tunable programs — each effect has parameters (colour palette, decay rate, sensitivity, spread, etc.). An LLM could:
  1. Generate *new* effect code with parameterised knobs
  2. Define the parameter search space based on audio characteristics
  3. The metric pipeline scores each candidate
  4. Evolutionary search finds optimal parameter sets
- **What it unlocks:** Composes with the entire metric pipeline as the fitness function. Distinct from ZeroGrads (which is gradient-based local optimisation) — this is global exploration of the parameter space and even the code space. Could discover entirely new effect structures, not just tune existing parameters.
- **Implementation complexity:** Medium — the LLM generates C++ effect code that compiles on ESP32. Main risk: LLM-generated code quality and compilation success rate.
- **Layer affected:** Optimisation (closed-loop strategy)

## MEDIUM COMBINATORIAL VALUE

### Sound2Hap — Audio-to-Physical-Output Perceptual Mapping (arXiv:2601.12245, Jan 2026)

- **Source domain:** Haptic rendering
- **The method:** CNN autoencoder that maps audio spectrograms to vibrotactile actuator signals, trained on human perceptual ratings (not physical ground truth). 34 participants rated 1,000 sound-vibration pairs. Uses 10ms windows with 50% overlap for real-time operation.
- **Combinatorial insight:** The architecture (audio spectrogram → CNN autoencoder → physical actuator output, optimised for human perception) is *structurally identical* to our pipeline (audio spectrogram → rendering engine → LED output, evaluated by quality metrics). Key transferable ideas:
  1. **Perceptual loss from human ratings** rather than signal-level comparison — train a loss function on how humans rate LED-audio pairings
  2. **10ms window / 50% overlap** processing cadence — validated for real-time perceptual quality
  3. **CNN autoencoder architecture** as the mapping function itself — could replace hand-coded audio-to-LED mapping
- **Implementation complexity:** Medium — the autoencoder architecture is straightforward. Collecting human ratings for LED-audio pairs is the bottleneck.
- **Layer affected:** Pipeline (audio-to-visual mapping) / L2 (audio-visual alignment)

## MEDIUM COMBINATORIAL VALUE

### Music2Latent2 — Summary Embeddings for Global Audio Context (arXiv:2501.17578, Jan 2025)

- **Source domain:** Audio compression / Music information retrieval
- **The method:** Compresses audio into *unordered* summary embeddings that capture global features (mood, energy, structural characteristics) rather than local temporal features. Outperforms ordered-sequence autoencoders on downstream MIR tasks.
- **Combinatorial insight:** All our current metrics operate at frame-level or short temporal windows. Summary embeddings provide a "holistic audio signature" that could enable:
  1. **Global mood matching:** Does the LED colour palette match the audio's mood embedding?
  2. **Structural awareness:** Does the LED visualisation reflect song structure (verse/chorus/bridge)?
  3. **Cross-song consistency:** Given similar audio embeddings, do we produce similar visualisation quality?
- **What it unlocks:** Composes with MLLM-as-judge (provide summary embedding as context for aesthetic evaluation) and with L4 aesthetic metrics (mood-appropriate colour/motion assessment).
- **Implementation complexity:** Medium — pre-trained Music2Latent2 models are available. Summary embedding extraction is a single forward pass.
- **Layer affected:** L2 (audio-visual alignment) / L4 (aesthetic)

## MEDIUM COMBINATORIAL VALUE

### musicolors — Computational Synesthesia for Audio-Visual Mapping (arXiv:2503.14220, Mar 2025)

- **Source domain:** Cross-modal perception / HCI
- **The method:** Implements computational chromesthesia — principled audio-to-colour mappings grounded in cross-modal perception research (pitch→lightness, timbre→hue, loudness→saturation correspondences derived from synesthesia and psychoacoustics studies).
- **Combinatorial insight:** Our current audio-to-colour mappings are engineering choices. Synesthesia research provides an empirically-grounded "ground truth" for what humans perceive as natural audio-visual correspondences. Could:
  1. **Validate** existing colour mappings against perceptual norms
  2. **Define L2 alignment metrics** — deviation from synesthetic mappings as a quality score
  3. **Provide perceptual priors** for the optimisation system — start from synesthetically natural mappings
- **Implementation complexity:** Low — the mapping functions are simple mathematical transformations.
- **Layer affected:** L2 (audio-visual alignment) / L4 (aesthetic)

## MEDIUM COMBINATORIAL VALUE

### Interactive Evolutionary Shader Generation (arXiv:2312.17587, SIGGRAPH 2024)

- **Source domain:** Procedural graphics / Interactive evolution
- **The method:** Interactive evolutionary algorithm integrated with Unity editor for exploring shader design spaces. Users provide aesthetic fitness judgements; the algorithm evolves shader parameters and structures.
- **Combinatorial insight:** LED effects are analogous to shaders — parametric visual programs. The interactive evolutionary approach provides a template for:
  1. **Human-in-the-loop evolution** of LED effects during development
  2. **Fitness function composition** — how to combine multiple quality metrics into a single evolutionary fitness score (weighted sum, Pareto front, etc.)
  3. **Design space exploration** — structured approach to discovering what the effect parameter space looks like
- **What it unlocks:** Composes with X-evolve (automated) and VisionReward (learned fitness) — provides the evolutionary framework, other outliers provide the fitness signal.
- **Implementation complexity:** Low-Medium — the evolutionary algorithm itself is standard. Integration with the metric pipeline is the main work.
- **Layer affected:** Optimisation (closed-loop strategy)

### EEG-Based Aesthetic Preference Decoding (Frontiers in Neuroergonomics 2024)

- **Source domain:** Neuroaesthetics / Passive BCI
- **The method:** Oscillatory EEG features decode visual aesthetic preference in ecologically valid settings. Consumer-grade 4-electrode EEG devices can decode selective attention to musical elements.
- **Combinatorial insight:** Long-term moonshot for ground truth collection. A passive BCI headset on a listener watching LED visualisations could provide continuous, implicit aesthetic preference data without requiring explicit ratings. This would be the highest-fidelity training signal for the VisionReward model.

The primary findings give us the metrics. The outliers give us what to optimise (group 1) and how to optimise (group 2).
