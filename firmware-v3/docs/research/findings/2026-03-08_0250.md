## 🔴 HIGH-VALUE FINDING

### APEX: Adaptive Priority Scheduling for Multi-Objective RL with Visual Quality
**arXiv:** [2601.06574](https://arxiv.org/abs/2601.06574) (January 2026)

**Relevance:** Solves the exact problem we'll face when combining L0-L4 metrics as reward signals — metrics at different scales, with different variances, producing conflicting gradient updates.

**Key transferable methods:**

1. **P³ Adaptive Priorities** — dynamically adjusts which objective receives emphasis during optimisation based on three factors: *learning potential* (how much improvement is still possible), *conflict penalty* (how much this objective fights others), and *progress need* (how far behind this objective is). For our system: if L2 (audio-visual sync) has high learning potential but conflicts with L4 (aesthetics), P³ schedules them appropriately rather than creating destructive gradient interference.

2. **Dual-Stage Adaptive Normalization** — solves "variance hijacking" where high-variance rewards (e.g., pixel-level L0 metrics with large dynamic range) implicitly dominate low-variance rewards (e.g., aesthetic scores on a 1-5 scale). Normalises each reward stream independently before combination. This is critical: without this, our L0 frame metrics would drown out L4 aesthetic scores simply due to variance differences.

3. **Gradient conflict detection** — identifies when objectives produce opposing update directions and adaptively reduces the weight of the conflicting objective. Prevents the optimiser from oscillating between "make it more colourful" (L4) and "reduce flicker" (L1) when they conflict.

**How we'd use it:** Wire L0-L4 metric scores as separate reward streams into an APEX-style multi-objective RL loop. APEX handles the reward normalisation, priority scheduling, and conflict resolution automatically. This replaces hand-tuned reward weighting (which is brittle and changes with every metric addition).

**Implementation complexity:** Medium — the P³ scheduling and Dual-Stage Normalization are algorithmic additions to the RL training loop, not separate models. Requires implementing the priority computation alongside the optimiser.

**Layer affected:** Optimisation (reward combination strategy for closed-loop system)

## 🔴 HIGH-VALUE FINDING

### ColorVideoVDP + CameraVDP: Perceptual Display Quality Assessment Ecosystem
**ColorVideoVDP:** [arXiv:2401.11485](https://arxiv.org/abs/2401.11485) (January 2024, SIGGRAPH 2024)
**CameraVDP:** [arXiv:2509.08947](https://arxiv.org/abs/2509.08947) (September 2025, SIGGRAPH Asia 2025)
**Code:** [github.com/gfxdisp/ColorVideoVDP](https://github.com/gfxdisp/ColorVideoVDP)

**Relevance:** Complete pipeline from camera capture of LED output → perceptually-calibrated quality score, modelling chromatic spatiotemporal contrast sensitivity. Extends elaTCSF (luminance-only flicker thresholds) into full colour-aware temporal+spatial quality assessment with uncertainty estimation.

**Key transferable methods:**

1. **Chromatic spatiotemporal contrast sensitivity model** — models how human vision responds to colour changes over time, not just luminance changes. This is critical for LED strips: our WS2812 LEDs produce highly saturated colours, and flicker/transition visibility varies dramatically by colour channel (blue flicker is less visible than green flicker at the same modulation depth). elaTCSF only handles luminance; ColorVideoVDP handles the full chromatic domain.

2. **Cross-channel contrast masking** — models how activity in one colour channel (e.g., a red pulse) masks perception of changes in another channel (e.g., simultaneous green dimming). On our LED strip, colour transitions often involve multiple channels changing simultaneously. Masking determines which changes the viewer actually perceives vs which are invisible.

3. **CameraVDP capture pipeline** — HDR stacking, MTF inversion, vignetting correction, geometric undistortion, homography transformation, colour correction, all calibrated to model camera measurement uncertainty. Our capture pipeline will need exactly this: photograph the LED strip, correct for camera artefacts, then run perceptual evaluation. CameraVDP provides the complete chain.

4. **Uncertainty estimation** — quantifies confidence in the quality prediction. When the metric says "quality score: 0.7 ± 0.15", the optimiser can treat this differently than "0.7 ± 0.02". Essential for the closed-loop system to know when to trust metric feedback vs when measurement noise dominates.

**How we'd use it:** Replace the ad-hoc combination of VBench temporal_flickering + SSIM + elaTCSF with a single, perceptually-calibrated VDP metric that handles luminance, chrominance, temporal, and spatial quality in one pass. CameraVDP provides the capture pipeline; ColorVideoVDP provides the quality predictor. The uncertainty estimate feeds into the optimiser's exploration/exploitation balance.

**Implementation complexity:** Medium — open-source PyTorch implementation exists. Main work is calibrating for our specific camera + LED strip geometry + viewing conditions. The display photometric model needs our WS2812 spectral characteristics.

**Layer affected:** L0 (frame), L1 (temporal), L3 (perceptual) — unified replacement for multiple metrics

## Finding 3: MORSE — Automatic Reward Shaping from Multiple Heuristic Rewards
**arXiv:** [2512.15120](https://arxiv.org/abs/2512.15120) (December 2025)

**Relevance:** Automates the hardest design decision in our closed-loop system — how to combine L0-L4 metric scores into a single reward signal that actually drives the optimiser toward better visualisations.

**Key transferable method:** Bi-level optimisation framework:
- **Inner loop:** trains a policy (our parameter optimiser) to maximise the current shaped reward (combined L0-L4 score)
- **Outer loop:** updates the reward combination function to optimise actual task performance (overall visualisation quality)
- **Stochastic exploration:** injects noise guided by task performance to avoid local optima in reward space

The key insight: rather than hand-tuning weights (e.g., "L2 sync is 3× more important than L4 aesthetics"), MORSE *learns* the reward combination that produces the best actual outcomes. The outer loop effectively searches over all possible reward weightings to find the one that, when optimised, produces the best visualisations.

**How we'd use it:** Define L0-L4 metrics as heuristic reward functions. Let MORSE's bi-level optimisation learn how to combine them. The inner loop tunes LED parameters; the outer loop tunes reward weights. This is complementary to APEX: APEX handles the gradient-level conflicts during optimisation; MORSE handles the meta-level question of "what should the reward even be?"

**Implementation complexity:** High — bi-level optimisation is computationally expensive (two nested training loops). However, the outer loop can run offline periodically while the inner loop runs in real-time. The stochastic exploration component is straightforward to implement.

**Layer affected:** Optimisation (meta-level reward design)

## Finding 4: Deep Time Warping — Differentiable Multi-Series Alignment
**arXiv:** [2502.16324](https://arxiv.org/abs/2502.16324) (February 2025)

**Relevance:** Extends TS3IM toward a more powerful alignment metric — aligns multiple time series simultaneously with a differentiable loss, enabling direct use as an L2 metric in the optimisation loop.

**Key transferable method:** Deep convolutional network that:
1. Decomposes signals into piecewise-linear sections at varying complexity levels
2. Enforces warping constraints: boundary, monotonicity, continuity (physically meaningful for temporal alignment)
3. Aligns **all signals simultaneously** rather than pairwise (critical: we need to align LED output with energy envelope AND spectral centroid AND onset strength AND beat activation simultaneously, not in separate pairwise comparisons)
4. Provides a **differentiable loss function** — gradient flows through the alignment, enabling end-to-end optimisation

**How we'd use it:** Feed the optimiser with a Deep Time Warping loss computed between:
- LED brightness time series (320 channels)
- Audio energy envelope
- Spectral centroid trajectory
- Onset strength function
- Beat activation function

The simultaneous multi-series alignment captures cross-modal temporal relationships that pairwise metrics (like corr(|Δ|) between just energy and brightness) miss. The differentiable loss means the optimiser gets gradients pointing toward "change LED parameters to better align with ALL audio features at once."

**Implementation complexity:** Medium — deep CNN architecture is standard. Main adaptation: our time series are very long (120 FPS × minutes), so the piecewise-linear decomposition at multiple scales is essential for tractability.

**Layer affected:** L2 (audio-visual alignment) — differentiable multi-feature alignment metric

## Queries That Yielded No Novel Findings

- **Reference-free AV alignment via contrastive learning** — CAV-MAE Sync (arXiv:2505.01237) provides frame-level audio-visual alignment, but trained on natural video. Too distant from LED strip domain without fine-tuning, and the core insight (frame-level > global alignment) is already implicit in our BeatAlign approach.
- **Beat confidence calibration / uncertainty** — no MIR-specific papers found. General uncertainty calibration research (UniCR, etc.) doesn't address audio-specific challenges.
- **Shapelet-based time series similarity** — ShapeFormer (arXiv:2405.14608) uses "Perceptually Important Points" for shapelet discovery. Interesting concept but classification-oriented, not quality-assessment oriented. No clear transferable metric.
- **Knowledge distillation for edge anomaly detection** — unified methods on edge (arXiv:2407.02968) target NVIDIA Jetson, not ESP32-S3 class devices. The quantisation approach (QAT/PTQ) is standard; no novel method for our constraint level.
- **LED colour appearance / gamut mapping** — no research combining CIECAM-family models with addressable LED strips. The closest work is the already-documented LED luminance paper (arXiv:2512.23051). ColorVideoVDP (above) subsumes the colour perception question.
- **Music structure-aware features** — SoniDo (arXiv:2411.01135) provides hierarchical audio features but overlaps with BeatFM (already documented). Insufficient differentiation.
- **Audio-reactive visual generation** — Network Bending (arXiv:2406.19589) is a generation technique, not an evaluation metric. No transferable quality assessment method.
