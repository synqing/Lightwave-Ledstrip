## 🔴 HIGH-VALUE FINDING: CP Loss — Learnable Channel-wise Perceptual Loss for 1D Time Series

**Paper:** CP Loss: Channel-wise Perceptual Loss for Time Series Forecasting
**arXiv:** [2601.18829](https://arxiv.org/abs/2601.18829) (January 2026, accepted ICASSP 2026)

**Relevance:** Directly fills the documented gap ("no concrete transferable method for visual 1D signals" — noted in 0833 and 0233 sweeps). Unlike TS3IM (which adapts SSIM's structural decomposition to time series) or MILO (which is image-only), CP Loss *learns* a per-channel perceptual space adapted to each channel's dynamics via hierarchical multi-scale decomposition. This is purpose-built for multi-channel 1D signals — exactly our 320-pixel LED strip.

**Key transferable method:**

1. **Hierarchical multi-scale decomposition per channel:** The signal is passed through a cascade of {2× downsample → learnable 1D convolution → 2× upsample} operations. Each stage produces a low-pass filtered version; subtraction isolates the high-frequency residual at that scale. This creates a multi-resolution feature pyramid for each channel independently.

2. **Per-channel perceptual space:** Loss is computed across these multi-scale representations rather than in raw signal space. The learnable convolution filters are optimised jointly with the main model, meaning the perceptual space adapts to each channel's characteristics (sharp transients vs smooth trends).

3. **Channel-agnostic loss vs channel-aware loss:** The paper demonstrates that standard MSE/MAE apply a uniform metric across all channels, failing to capture channel-specific dynamics like sharp fluctuations or trend shifts. CP Loss handles this explicitly.

**How we'd use it:**

- **L0/L3 metric replacement:** For comparing predicted vs actual LED frame sequences, treat each pixel (or pixel group/segment) as a "channel" in a multivariate time series. Compute CP Loss across all 320 channels at multiple temporal scales. This gives us a learned perceptual distance that respects per-pixel dynamics (e.g., a fast-flickering pixel on a beat vs a slow-fading background pixel have different perceptual spaces).

- **Closed-loop optimisation loss:** Because CP Loss is differentiable and learnable, it can serve as the objective function in our parameter optimisation loop (composing with ZeroGrads from the 0800 sweep).

- **Multi-scale temporal sensitivity:** The hierarchical decomposition naturally captures both frame-level changes (fast scale) and longer temporal patterns (slow scale), aligning with our L0 (frame) and L1 (temporal) metric layers.

**Key advantage over TS3IM:** TS3IM uses fixed decomposition (trend/variability/structure); CP Loss uses *learned* decomposition adapted per channel. For LED strips where different pixel regions serve different visual functions (edge pixels, centre region, highlight zone), learned per-channel decomposition is materially better.

**Implementation complexity:** Low-Medium — the core is a few 1D convolution layers per channel. No large model training. Can be implemented in PyTorch with ~50 lines, then converted to a metric for the evaluation pipeline. Joint optimisation with the main model requires integrating into the training loop.

**Layer affected:** L3 (perceptual similarity) — primary. Also L0 (frame quality) and L1 (temporal coherence) via multi-scale decomposition.

## Finding 2: TAVCE CAR Loss — Temporal Audio-Visual Correlation Alignment

**Paper:** Exploiting Temporal Audio-Visual Correlation Embedding for Audio-Driven One-Shot Talking Head Animation
**arXiv:** [2504.05746](https://arxiv.org/abs/2504.05746) (April 2025, accepted TMM 2025)

**Relevance:** Introduces a *different* approach to audio-visual alignment than existing methods in our stack. BeatAlign correlates beat timestamps with visual change peaks (discrete events). corr(|Δ|) correlates absolute first-order differences (overall envelope correlation). CycleSync measures reconstruction ability (can you recover audio from video?). CAR Loss enforces a **structural property**: if two adjacent audio clips have a certain temporal relationship, their corresponding visual frames must have the same temporal relationship.

**Key transferable method:**

**Correlation-Aware Regularization (CAR) Loss:**
1. Compute temporal similarity between adjacent audio clip pairs (e.g., cosine similarity of learned audio features for window t and window t+1).
2. Compute temporal similarity between corresponding adjacent visual frame pairs.
3. Minimise the discrepancy between audio temporal similarity and visual temporal similarity.

This enforces the constraint: "visually similar adjacent moments should correspond to sonically similar adjacent moments, and vice versa." It's a *second-order* alignment metric — it doesn't align individual events (like BeatAlign) but aligns the *pattern of change* between audio and visual domains.

**How we'd use it:**

- **L2 metric (audio-visual alignment):** For each sliding window pair over the LED capture + audio, compute audio feature similarity (spectral centroid, energy, MFCC) and LED frame similarity (e.g., via CP Loss from Finding 1 or simple cosine). CAR Loss = |audio_sim - visual_sim|. Low CAR means the LED animation is tracking the audio's temporal structure, not just matching individual beats.

- **Captures sub-beat granularity:** Because it operates on continuous temporal windows rather than discrete beat events, it naturally captures the sub-beat dynamics we identified as a gap in Domain 1 (onset-to-visual latency, timbral shifts between beats).

- **Complements BeatAlign:** BeatAlign captures "are beats visually marked?" CAR captures "does the visual flow match the audio flow between beats?" Together they cover both discrete and continuous audio-visual alignment.

**Implementation complexity:** Low — it's a loss function computed over windowed feature similarities. No model training required (uses pretrained features or hand-crafted audio features). Core computation is cosine similarity + L1/L2 difference.

**Layer affected:** L2 (audio-visual alignment) — continuous temporal alignment

## Finding 3: Meta Audiobox Aesthetics — Open-Source Audio Quality Predictor

**Paper:** Meta Audiobox Aesthetics: Unified Automatic Quality Assessment for Speech, Music, and Sound
**arXiv:** [2502.05139](https://arxiv.org/abs/2502.05139) (February 2025)
**Code:** [github.com/facebookresearch/audiobox-aesthetics](https://github.com/facebookresearch/audiobox-aesthetics)

**Relevance:** Provides an off-the-shelf, no-reference audio quality predictor that scores any audio (speech, music, sound) along 4 decomposed axes. Different from PrismAudio (which is a reward framework for AV generation). This is a pretrained inference-only tool for assessing the *input audio quality* feeding our evaluation pipeline.

**Key transferable method:**

1. **4-axis audio aesthetic decomposition:** Decomposes subjective audio quality into 4 independent axes (specific axis definitions not fully detailed in abstract, but the trained predictors produce per-axis scores).
2. **No-reference, per-item prediction:** Works on arbitrary audio clips without a reference signal. Released as open-source pretrained models.
3. **Cross-domain robustness:** Trained on speech, music, and sound datasets, so it handles music input reliably.

**How we'd use it:**

- **Pipeline quality gate:** Score input audio quality before running evaluation metrics. Low audio aesthetic score → flag the evaluation as unreliable (garbage in → garbage out). This prevents the closed-loop optimiser from chasing phantom quality issues caused by poor source audio.
- **Audio quality as confound variable:** Include Audiobox aesthetic scores as covariates when analysing L2-L4 metric distributions. This lets us distinguish "bad visualisation of good music" from "any visualisation of bad music."

**Implementation complexity:** Low — pretrained model, pip-installable, inference-only. No training required.

**Layer affected:** Pipeline (audio quality assessment / pre-filter)

**Assessment:** CP Loss is the standout finding this sweep. It fills the most prominent documented gap (1D perceptual similarity) with a method that is purpose-built for multi-channel 1D signals, learnable, differentiable, and lightweight. Combined with ZeroGrads (0800 sweep) and PrismAudio's decomposed reward framework (0833 sweep), it closes the loop: CP Loss provides the per-channel perceptual objective → ZeroGrads provides the gradient → PrismAudio provides the decomposed multi-dimensional reward structure.

The TAVCE CAR Loss provides a complementary L2 metric that captures *continuous* audio-visual alignment (vs BeatAlign's *discrete* event alignment), addressing the sub-beat granularity gap identified in Domain 1.
