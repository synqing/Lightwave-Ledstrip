## Finding 1: VisJudge — Specialised MLLM for Data Visualisation Quality Assessment

**Paper:** VisJudge-Bench: Aesthetics and Quality Assessment of Visualisations (arXiv:2510.22373, Oct 2025 / updated 2026)

**One-sentence relevance:** First benchmark and specialised MLLM for evaluating *data visualisation* quality across three dimensions — Fidelity, Expressiveness, Aesthetics — directly applicable because LED strip patterns ARE data visualisations (audio features mapped to visual output).

**Key transferable method:**
- Three-dimensional evaluation rubric adapted for LED visualisation:
  - **Fidelity** = does the LED pattern faithfully represent the audio structure? (beats visible, frequency bands mapped correctly)
  - **Expressiveness** = can a viewer perceive musical features from the LED output? (rhythm, dynamics, timbral changes)
  - **Aesthetics** = colour harmony, motion smoothness, visual appeal
- The specialised VisJudge model showed **23.9% MAE reduction** over GPT-5 on visualisation assessment (MAE 0.421 vs 0.553) and **60.5% improvement** in human correlation (0.687 vs 0.428)
- Key implication: domain-specific fine-tuning of MLLM judges massively outperforms general-purpose models. We should fine-tune our MLLM judge on LED visualisation examples rather than using vanilla GPT/Claude.
- Dataset of 3,090 expert-annotated samples across 32 chart types provides a template for building our own LED visualisation quality dataset.

**Implementation complexity:** Medium — requires capturing LED video, framing evaluation prompts with the three-dimensional rubric, running against an MLLM API. Building a domain-specific training set is the hard part.

**Which layer it affects:** L4 (aesthetic evaluation)

## Finding 2: Automated Rubric Generation (Rubric-ARM / RRD / Autorubric)

**Papers:**
- Rubric-ARM (arXiv:2602.01511, Feb 2026)
- RRD: Rethinking Rubric Generation (arXiv:2602.05125, Feb 2026)
- Autorubric: A Unified Framework for Rubric-Based LLM Evaluation (arXiv:2603.00077, Mar 2026)

**One-sentence relevance:** Three concurrent papers on automatically generating and refining evaluation rubrics for LLM/MLLM judges — solving the problem that our hand-crafted aesthetic rubrics are brittle and may miss important quality dimensions.

**Key transferable methods:**
- **Rubric-ARM:** Jointly optimises a rubric generator and a judge via RL from preference feedback. Treats rubric generation as a *latent action* learned to maximise judgement accuracy. Could learn what evaluation criteria best discriminate good vs. bad LED visualisations from human preference data.
- **RRD:** Recursive decompose-filter cycle that breaks coarse criteria into fine-grained discriminative sub-criteria. Starting from "good music visualisation," it would auto-discover dimensions like "bass response speed," "colour transition smoothness," "beat emphasis intensity" without us hand-specifying them.
- **Autorubric:** Provides standardised infrastructure: criteria specification + performance-level descriptions along a defined continuum. Template for structuring our evaluation pipeline.

**Implementation complexity:** Medium-High — Rubric-ARM requires collecting preference data (human A/B ratings of LED visualisation quality pairs). Could bootstrap from synthetic preferences generated by existing metrics (e.g., generate pairs where Metric A ranks differently from Metric B, have humans arbitrate).

**Which layer it affects:** L4 (aesthetic evaluation), Optimisation (rubric refinement is itself a meta-optimisation loop for the evaluation system)

## Finding 3: VABench Metric Suite — Specific Implementation Patterns

**Paper:** VABench: A Comprehensive Benchmark for Audio-Video Generation (arXiv:2512.09299, Dec 2025)

**One-sentence relevance:** Documents 15 fine-grained AV evaluation metrics with specific model choices and formulas, several of which provide concrete implementation patterns beyond what we've previously captured.

**Key transferable methods (NEW details not previously documented):**

1. **AudioAesthetic** — evaluates audio production quality via Audiobox across four dimensions: Content Enjoyment (CE), Content Usefulness (CU), Production Complexity (PC), Production Quality (PQ). Formula: `(CE + CU + PQ - PC) / 4`. Could be used to classify audio input quality tier, adjusting evaluation thresholds accordingly (a lo-fi track should be evaluated differently from a studio master).

2. **Text-Audio Align via CLAP** — cosine similarity in CLAP embedding space between audio and text descriptions. Transferable: describe the *intended* visualisation style in text ("energetic red pulses on bass hits, flowing blue waves on melody"), compute CLAP-style alignment between that description and the actual audio, then chain with visual evaluation.

3. **MLLM-based Macro metrics** — VABench structures MLLM evaluation into five distinct dimensions (Alignment, Artistry, Expressiveness, Audio Realism, Visual Realism) with separate scoring for each. This decomposition is more granular than our current single-score MLLM judge approach.

4. **Micro QA pairs** — 3-7 custom yes/no questions per sample targeting specific physical properties. For LED evaluation: "Does LED brightness increase during the drum fill at 0:23?" "Do the blue pixels track the melodic line?" This micro-question approach could provide more reliable scoring than open-ended aesthetic judgement.

**Implementation complexity:** Low-Medium (individual metrics) to Medium (full pipeline)

**Which layer it affects:** L2 (Desync already logged), L4 (MLLM macro/micro metrics), Pipeline (AudioAesthetic for input classification)

# ⚠️ POST-HOC CORRECTIONS — 2026-03-08

**Added after review. The findings above contain structural errors that would corrupt downstream decision-making if consumed uncorrected. Read this section before acting on anything above.**

## Problem 1: Taxonomy Failure

The original findings mix four different species of thing as if they were comparable:

| Finding | Actual type | Originally presented as |
|---------|------------|------------------------|
| VisJudge | `BENCHMARK` — chart quality evaluation benchmark | "Directly applicable metric" |
| Rubric-ARM / RRD / Autorubric | `META-FRAMEWORK` — rubric generation infrastructure | "Finding" alongside direct metrics |
| VABench metrics | `BENCHMARK` — AV generation evaluation suite | "Implementation patterns" |
| Syncphony Motion-aware Loss | `TRAINING-METHOD` — generation loss function | "Metric weighting strategy" |

**Correction:** These are not interchangeable. A metric tells you a number about your output. A judge tells you an opinion. A benchmark tells you how to compare judges. A training loss tells you how someone else optimised their generator.

## Problem 2: Transfer Overstatements

### VisJudge — DOWNGRADED
**Original claim:** "LED strip patterns ARE data visualisations — directly applicable."
**Correction:** This is [HYPOTHESIS] presented as [FACT]. VisJudge evaluates charts and dashboards where "fidelity" means data encoding correctness. LED music visualisation is an expressive mapping where multiple valid interpretations exist for the same audio. VisJudge's three-dimensional rubric (Fidelity/Expressiveness/Aesthetics) is *inspiration* for rubric design, not a transplantable evaluation system.
**Revised status:** DEFER — rubric dimension inspiration only. Not actionable until our own evaluation rubric exists.

### AudioAesthetic — DOWNGRADED
**Original claim:** "Could be used to classify audio input quality tier."
**Correction:** AudioAesthetic evaluates *generated* audio production quality, not input audio characteristics. Using it to classify input tracks is a conceptual leap with no validation. Transfer confidence: Low.
**Revised status:** REJECT — wrong direction. We need audio *analysis* (spectral features, onset detection), not audio *quality scoring*.

### CLAP Text-Audio Alignment — DOWNGRADED
**Original claim:** "Describe intended visualisation style in text, compute CLAP alignment."
**Correction:** CLAP measures semantic agreement between audio content and text descriptions ("a dog barking" matches dog-bark audio). It does not measure whether a visualisation style spec is appropriate for an audio track. The proposed chain (text description → CLAP → visual evaluation) has two untested inference steps. Transfer confidence: Low.
**Revised status:** REJECT — conceptual chain too long, no validation path.

### Syncphony Motion-aware Weighting — CORRECTED
**Original claim:** "~10 lines of code" weighting strategy for existing metrics.
**Correction:** The Motion-aware Loss is a training objective for video generation. The idea of weighting evaluation metrics by audio energy is a [HYPOTHESIS] inspired by this loss, not a validated evaluation technique. Implementation may indeed be ~10 lines, but the evidence that energy-weighted sync scores correlate better with human judgement than unweighted scores is zero.
**Revised status:** DEFER — reasonable hypothesis, but needs A/B validation against unweighted baseline before adoption.
**Kill criterion:** Reject if energy-weighted BeatAlign/corr(|Δ|) does not improve rank correlation with human pairwise preferences over unweighted versions.
