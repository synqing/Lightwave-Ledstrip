## HIGH-VALUE FINDING

### VisionDirector: Vision-Language Guided Closed-Loop Refinement for Generative Image Synthesis
- **ArXiv ID:** [arXiv:2512.19243](https://arxiv.org/abs/2512.19243) (v3, updated **2026-02-05**)
- **Relevance:** Directly implements the planner→editor→verifier closed-loop architecture with rollback and efficiency optimisation via RL — this is the closest published analogue to our PROPOSE→APPLY→EVALUATE loop.
- **Key Transferable Methods:**
  1. **Micro-grid sampling with semantic verification:** Generate N candidate parameter modifications per proposal step (paper uses 4 T2I samples), score each against pending quality goals via a verifier, select the best. Maps to: generate multiple parameter mutation candidates, evaluate each against our L0–L4 metric stack, apply the winner.
  2. **Goal-level rollback:** When an applied modification degrades overall alignment score, revert to previous best state and reshuffle remaining goals. Maps to: if a parameter change worsens composite metric score, roll back and try alternative modifications. Prevents accumulated quality degradation across iterative tuning steps.
  3. **Group Relative Policy Optimization (GRPO) for proposal efficiency:** Uses RL (token-level masking, alignment-based reward, rendered rollouts) to train the proposal engine to generate better-aligned suggestions in fewer steps. Result: median editing rounds dropped 4.2 → 3.1 (26% latency reduction). Maps to: training/tuning our proposal engine to converge faster by learning which parameter modifications tend to succeed for given audio signatures.
  4. **0–5 alignment scoring with per-attribute breakdown:** Verifier scores each goal on a dense 0–5 scale across attributes (typography, lighting, spatial layout → for us: beat-sync, colour harmony, spectral mapping, temporal smoothness). Enables structured multi-dimensional quality assessment rather than a single composite score.
- **Implementation Complexity:** Medium — the architecture pattern is directly applicable; GRPO requires training data from our evaluation loop (achievable via logged runs). Rollback and multi-candidate sampling are straightforward engineering.
- **Target Layer:** Optimisation (closed-loop) + Architecture (system design)
- **Why This Is High-Value:** Our system currently lacks a formal rollback mechanism and multi-candidate evaluation per proposal step. VisionDirector's pattern of "generate multiple candidates → verify each → select best → rollback on regression" is exactly the robustness layer our PROPOSE→APPLY loop needs. The GRPO training loop for proposal efficiency is a longer-term but high-leverage addition.

## Near-Miss Findings (Outside 60-Day Window but Noteworthy)

### Video Consistency Distance (VCD) — Frequency-Domain Temporal Consistency
- **ArXiv ID:** [arXiv:2510.19193](https://arxiv.org/abs/2510.19193) (Oct 2025)
- **Why Near-Miss:** Defines temporal consistency in the frequency space of video frame features rather than pixel-space or perceptual-space. Distinct from VBench temporal_flickering (frame-pair SSIM) and WCS (learned sub-component weights). Could complement our L1 temporal coherence layer with a frequency-domain signal, potentially better at detecting periodic artefacts (e.g., flickering at specific Hz that correlates with beat frequency).
- **Status:** Outside window. Monitor for follow-up work or v2 updates.

### Perceptual Straightening Guidance (PSG) — Curvature Penalty in Perceptual Space
- **ArXiv ID:** [arXiv:2510.25420](https://arxiv.org/abs/2510.25420) (Oct 2025)
- **Why Near-Miss:** Based on the neuroscience perceptual straightening hypothesis — smooth temporal evolution should have low curvature in a learned perceptual space (DINOv2 features). The "Straightness" metric measures this curvature. Distinct from our known temporal metrics and could serve as a novel L1 signal: LED frame sequences that are "perceptually straight" = smooth transitions, high curvature = jarring/flickering.
- **Status:** Outside window. The underlying concept (trajectory curvature in feature space as a temporal coherence metric) is worth tracking.

### SAVGBench Spatial AV-Align Metric
- **ArXiv ID:** [arXiv:2412.13462](https://arxiv.org/abs/2412.13462) (updated Feb 2026)
- **Why Not Logged as Finding:** Within the window but not applicable. The metric measures spatial alignment between stereo sound source positions and visual object locations. Our system is mono audio → 1D LED strip — no spatial dimension to align. The YOLOX + SELD pipeline is irrelevant for our use case.
