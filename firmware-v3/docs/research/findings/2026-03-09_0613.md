## Finding 1: UrbanAlign — Post-hoc Concept-Bottleneck Calibration for VLM Judge Alignment Without Retraining

**Paper:** arXiv:2602.19442 — "UrbanAlign: Post-hoc Semantic Calibration for VLM-Human Preference Alignment" (February 23, 2026; revised March 5, 2026)
**Authors:** Yecheng Zhang, Rong Zhao, Zhizhou Sha, Yong Li, Lei Wang, Ce Hou, Wen Ji, Hao Huang, Yunshan Wan, Jian Yu, Junhao Xia, Yuru Zhang, Chunlei Shi

**One-sentence relevance:** Demonstrates that a frozen VLM can be aligned with human quality preferences through a training-free post-hoc calibration pipeline — directly applicable to calibrating our L4 MLLM-as-judge scores without model fine-tuning.

**Key transferable method:**

1. **Three-stage concept-bottleneck pipeline:**
   - **Stage 1 — Concept Mining:** Automatically extract interpretable evaluation dimensions from exemplar pairs (better/worse examples). For our system: present the MLLM with pairs of LED frames labelled as "good" vs "poor" quality, and let it mine the dimensions it uses to distinguish them (e.g., "colour diversity," "rhythmic responsiveness," "brightness distribution," "transition smoothness").
   - **Stage 2 — Multi-agent Structured Scoring:** An Observer-Debater-Judge chain extracts concept scores. The Observer describes the visual, the Debater challenges the description, the Judge produces a structured score per dimension. This reduces single-pass scoring variance by forcing adversarial verification within the scoring pipeline.
   - **Stage 3 — Geometric Calibration:** Locally-weighted ridge regression on a hybrid manifold maps raw concept scores to calibrated human preference labels. The "locally-weighted" aspect means the calibration adapts to neighbourhood structure — different quality ranges get different calibration functions. This is critical: our quality landscape is highly non-linear (the difference between "mediocre" and "good" is perceptually larger than "good" to "excellent").

2. **Performance uplift:** +16.3 percentage points over uncalibrated VLM scoring, +15.1pp over the best supervised baseline. Achieving this with ZERO weight modification means it works with any VLM — including closed-source API models like GPT-4V or Claude.

3. **Dimension optimisation loop:** The pipeline includes an end-to-end loop that optimises WHICH concepts to mine based on calibration accuracy. Concepts that don't contribute to calibrated score accuracy are pruned. This is a meta-learning step that discovers the minimal interpretable concept set for quality evaluation.

**Concrete application to our system:**

- **L4 calibration protocol:** Our current L4 approach uses MLLM-as-judge with CyclicJudge rotation (prompt variant cycling). UrbanAlign adds a NEW layer: take the raw MLLM judge outputs, decompose them into interpretable concept scores, and calibrate those scores against a small labelled preference dataset. The workflow:
  1. Collect 50–100 paired LED frame outputs with human quality labels (better/worse).
  2. Run concept mining to discover the MLLM's natural evaluation dimensions for LED visualisation quality.
  3. Build the Observer-Debater-Judge chain for structured per-concept scoring.
  4. Fit locally-weighted ridge regression to calibrate concept scores → composite quality score.
  5. At evaluation time: raw MLLM judge → concept extraction → calibrated composite score.

- **Composability with existing L4 methods:** This slots between CyclicJudge (which handles prompt-level bias) and null-space projection (which handles activation-level bias, from 2026-03-10_0110.md). The pipeline becomes: CyclicJudge rotates prompt variants → MLLM generates raw evaluation → concept extraction decomposes into structured dimensions → geometric calibration maps to human-aligned scores. Each stage addresses a different failure mode.

- **Interpretability advantage:** Unlike end-to-end scoring, the concept-bottleneck architecture tells us WHY a frame scored poorly. If the "transition smoothness" concept scores low but "colour diversity" scores high, the optimiser can target transition parameters specifically. This bridges L4 (aesthetic) evaluation directly into the PROPOSE phase — the optimiser gets actionable diagnostic information, not just a scalar score.

- **Observer-Debater-Judge for variance reduction:** The three-agent scoring chain is particularly relevant because single-pass MLLM evaluation has high variance (documented noise floor of 4–16% from null-space projection finding). The Debater forces the Observer to justify scores, catching hallucinated quality assessments. This is structurally different from averaging multiple MLLM calls — it introduces adversarial verification within a single evaluation.

**Implementation complexity:** Medium. Concept mining and Observer-Debater-Judge chain are prompt engineering tasks (no model training). Geometric calibration requires a small labelled dataset (50–100 pairs) and standard ridge regression. The end-to-end loop requires iteration but is a one-time setup cost.

**Target layer:** L4 (aesthetic evaluation — judge calibration and structured concept decomposition)

**Differentiation from existing findings:**
- CyclicJudge (2026-03-09_2341.md): prompt-level bias rotation → UrbanAlign: score-level calibration. Different layers of the evaluation stack.
- Null-space projection (2026-03-10_0110.md): removes known biases from model activations → UrbanAlign: calibrates concept scores to human preferences. Complementary.
- FlowPortrait regulariser (2026-03-09_2310.md): adds hard metric regularisers to MLLM scores → UrbanAlign: decomposes MLLM scores into calibratable concepts. Can be combined: hard metrics as gates (FlowPortrait/ToolRLA), concept-calibrated MLLM scores as quality gradient.

## Finding 2: CP Loss — Channel-wise Perceptual Loss for Multi-Channel Time Series

**Paper:** arXiv:2601.18829 — "CP Loss: Channel-wise Perceptual Loss for Time Series Forecasting" (January 25, 2026)
**Authors:** Yaohua Zha, Chunlin Fan, Peiyuan Liu, Yong Jiang, Tao Dai, Hai Wu, Shu-Tao Xia
**Accepted at:** ICASSP 2026

**One-sentence relevance:** Introduces a learnable channel-wise perceptual loss specifically designed for multi-channel sequential signals — the first perceptual quality metric purpose-built for the data shape of our 320-pixel × 3-channel (RGB) LED output at 120 FPS.

**Key transferable method:**

1. **Channel-wise learnable filter:** Instead of applying a single perceptual loss function across all channels (as MSE, SSIM, or LPIPS do), CP Loss learns a separate decomposition filter per channel. Each filter decomposes the raw 1D signal into disentangled multi-scale representations. For our RGB LED strip: the R, G, and B channels each get their own perceptual space, reflecting the fact that perceptual sensitivity differs across colour channels (human vision is more sensitive to luminance/green than chrominance/blue).

2. **Multi-scale signal decomposition:** The learnable filter produces representations at multiple temporal scales. For our 120 FPS signal, this would capture:
   - Fine scale: frame-to-frame micro-transitions (single LED changes)
   - Medium scale: beat-aligned colour sweeps (4–8 frame patterns)
   - Coarse scale: section-level colour palette evolution (100+ frame patterns)
   This multi-scale decomposition is exactly what our L1 (temporal coherence) and L3 (perceptual similarity) layers need — a way to evaluate quality at different temporal granularities simultaneously.

3. **Joint optimisation with task objective:** The perceptual space is optimised jointly with the downstream task. In our case: the filter would be trained to produce perceptual spaces that correlate with human quality preferences for LED visualisation. The filter learns which temporal patterns matter for quality and which are perceptually irrelevant — a data-driven alternative to hand-crafted temporal frequency analysis.

**Concrete application to our system:**

- **L3 perceptual similarity replacement:** Our current L3 layer uses standard metrics (DreamSim, LPIPS, Wasserstein distance) designed for 2D images. These require either: (a) treating each frame as a tiny 320×1 "image" (degenerate case, poor metric behaviour), or (b) stacking frames into a 2D representation (artificial spatial structure). CP Loss provides a native 1D multi-channel perceptual loss that operates directly on our signal's natural shape: 320 pixels × 3 channels × T frames. No reshaping or dimensionality hacks needed.

- **Multi-scale temporal evaluation:** The multi-scale decomposition directly maps to our evaluation needs:
  - **Scale 1 (1–4 frames):** Flicker detection. Rapid oscillations that appear as visual noise.
  - **Scale 2 (4–32 frames):** Beat-aligned response. Do colour transitions align with musical beats?
  - **Scale 3 (32–256 frames):** Phrase-level coherence. Does the overall palette evolve sensibly across musical phrases?
  - Each scale contributes independently to the composite perceptual score, giving the optimiser actionable information about WHICH temporal scale has quality problems.

- **Training data requirement:** The filter requires training data: pairs of (LED output, quality label). This can be generated from our existing capture pipeline — record LED output for various parameter configurations, label quality (human ratings or proxy labels from L4). The filter trains quickly (lightweight architecture) and then serves as a fast L3 evaluator at inference time.

- **Interaction with L1 temporal coherence:** Our L1 layer currently uses VBench temporal_flickering and World Consistency Score — both designed for video. CP Loss's multi-scale decomposition subsumes L1: the fine-scale component IS a flicker detector, but learned rather than hand-crafted. This could either replace L1 or serve as a learned complement to the hand-crafted L1 metrics.

**Implementation complexity:** Medium. Requires: (a) adapting CP Loss architecture from time-series forecasting to quality evaluation (change the downstream task head), (b) collecting training pairs (LED output + quality labels), (c) training the channel-wise filters (lightweight — minutes on a single GPU). Inference is fast (learnable convolution filters).

**Target layer:** L3 (perceptual similarity) + potential L1 (temporal coherence) subsumption

**Differentiation from existing findings:**
- DreamSim/LPIPS (documented in "What We Already Know"): 2D image metrics repurposed for our signal → CP Loss: native 1D multi-channel metric. Structurally different.
- Wasserstein Distance (documented): distribution-level comparison → CP Loss: temporal-structure-aware perceptual comparison. Different granularity.
- VBench temporal_flickering (documented): hand-crafted temporal consistency → CP Loss: learned multi-scale temporal quality. Data-driven alternative.
