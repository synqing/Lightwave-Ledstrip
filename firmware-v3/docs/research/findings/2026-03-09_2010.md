## Finding 1: PSNR_DIV — Motion-Divergence Weighted Temporal Quality Metric

**Paper:** arXiv:2510.01361 — "An Efficient Quality Metric for Video Frame Interpolation Based on Motion-Field Divergence" (Oct 2025, updated Jan 2026)
**Authors:** Conall Daly, Darren Ramsook, Anil Kokaram

**One-sentence relevance:** A computationally cheap temporal quality metric that weights per-pixel quality by optical flow divergence, originally developed for archival film restoration to detect temporal inconsistencies — directly applicable to our L1 temporal coherence layer where standard SSIM/PSNR ignore motion dynamics.

**Key transferable method:**
- Computes optical flow divergence map between consecutive frames.
- Uses divergence as a spatial weighting mask on top of PSNR (or any per-pixel metric): regions with high divergence (motion boundaries, flickering) receive higher weight in the quality score.
- Achieves +0.09 Pearson LCC over FloLPIPS while being **2.5× faster** and using **4× less memory**.
- Originated from archival film restoration where it detected temporal artefacts (flicker, judder, splice defects).

**Concrete application to our system:**
- Our LED output at 120 FPS has both intentional motion (beat-triggered transitions, sweeps, pulses) and unintentional artefacts (flicker, jitter between frames).
- PSNR_DIV's divergence weighting would automatically focus quality assessment on exactly the regions/frames where motion is happening — which is where both the best and worst visualisation behaviour occurs.
- For a 320×1 LED strip, optical flow reduces to 1D displacement estimation between consecutive frames, which is computationally trivial.
- The divergence weighting could be combined with our existing frame-to-frame consistency metric to create a motion-aware variant that penalises unintentional flicker more heavily than intentional transitions.
- Extremely cheap to compute: suitable for real-time evaluation within the capture loop.

**Implementation complexity:** Low — 1D optical flow + divergence computation on 320-pixel frames at 120 FPS is negligible. The weighting scheme is a simple element-wise multiplication with any existing pixel-wise quality metric.

**Target layer:** L1 (temporal coherence) — enhances existing frame-to-frame consistency metrics with motion-awareness.

**Differentiation from existing findings:** No previous finding has proposed using optical flow divergence as a quality weighting mechanism. VBench temporal_flickering and World Consistency Score use optical flow for motion detection but not as a quality-weighting mask. This is a distinct, complementary approach.

## Finding 2: VisionDirector — Goal-Level Closed-Loop Verification with Rollback

**Paper:** arXiv:2512.19243 — "VisionDirector: Vision-Language Guided Closed-Loop Refinement for Generative Image Synthesis" (Dec 2025, Feb 2026 revision)
**Authors:** (Multiple — see paper)

**One-sentence relevance:** Implements a training-free closed-loop supervisor that decomposes complex quality requirements into structured goals, tracks per-goal state across iterations, and performs semantic verification with rollback — an architectural pattern directly transferable to our multi-metric PROPOSE→APPLY→EVALUATE loop.

**Key transferable method:**
- **Structured goal extraction:** Long/complex quality requirements are decomposed into individual, independently verifiable goals (e.g., "colour harmony", "beat alignment", "spatial balance").
- **Per-goal state tracking:** Each goal has its own satisfaction state across iterations. The system knows which goals are met and which are not after each modification.
- **Micro-grid sampling:** When proposing modifications, multiple candidates are generated and evaluated per goal.
- **Semantic verification + rollback:** After each modification, a VLM verifies whether the goal was actually improved. If not, the modification is rolled back before proceeding.
- **Dynamic strategy selection:** The system decides whether a goal needs one-shot generation (complete re-parameterisation) vs. staged editing (incremental adjustment).
- **Goal-level rewards:** Each goal emits its own reward signal, enabling fine-grained optimisation rather than optimising a single composite score.

**Concrete application to our system:**
- Our L0–L4 metric layers already decompose quality into layers. VisionDirector's pattern goes further: within each layer, individual metrics become independent "goals" with their own state.
- After a parameter modification (e.g., increasing decay rate), each metric/goal is re-evaluated independently. If decay-rate change improved temporal smoothness (L1) but degraded beat alignment (L2), the system can rollback the change for the L2-affecting parameters while keeping the L1 improvement.
- This prevents the current risk of composite-score optimisation masking per-metric regressions.
- The "dynamic strategy selection" maps to our system: some quality issues need complete palette changes (one-shot) while others need incremental brightness adjustments (staged edit).
- Training-free: the architecture is a supervisor wrapping existing components, requiring no model training.

**Implementation complexity:** Medium — requires implementing per-metric state tracking and a rollback mechanism in the parameter space. The architecture is well-specified in the paper. Main engineering challenge: defining which parameter subsets affect which metrics (the goal→parameter mapping).

**Target layer:** Optimisation (closed-loop strategy) + Architecture (system design pattern)

**Differentiation from existing findings:**
- TALBO (2026-03-09_0900.md) addresses non-stationary objectives via temporal awareness.
- BoGA (2026-03-09_0900.md) addresses proposal diversity via GA + surrogate filtering.
- VisionDirector addresses a different problem: per-goal verification and rollback within the optimisation loop. These three are complementary architectural patterns that could be combined: TALBO for temporal adaptation, BoGA for diverse proposals, VisionDirector for structured goal-level verification with rollback.
