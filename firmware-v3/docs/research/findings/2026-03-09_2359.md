## Finding 1: REACT — Flow-Magnitude-Guided Dynamic Frame Sampling for Efficient Video Quality Evaluation

**Paper:** arXiv:2601.04033 — "Thinking with Frames: Generative Video Distortion Evaluation via Frame Reward Model" (January 7, 2026)
**Authors:** Yuan Wang, Borui Liao, Huijuan Huang, Jinda Lu, Ouxiang Li, Kuien Liu, Meng Wang, Xiang Wang

**One-sentence relevance:** Introduces FMG-DFS (Flow-Magnitude-Guided Dynamic Frame Sampling) — a method that selects evaluation-worthy frames by computing optical flow magnitudes and sampling at motion intensity peaks, directly applicable to reducing evaluation cost in our 120 FPS metric pipeline.

**Key transferable method:**

1. **FMG-DFS (Flow-Magnitude-Guided Dynamic Frame Sampling):** Instead of evaluating every frame, compute dense optical flow between consecutive frames, extract flow magnitudes per frame, and sample frames at peaks of motion intensity. These are the frames most likely to contain visual artefacts or quality-relevant events. The remaining frames are skipped.

2. **Two-stage frame-level reward model training:** (a) Supervised fine-tuning with masked loss for domain knowledge injection, then (b) reinforcement learning via Group Relative Policy Optimization (GRPO) with pairwise rewards to align frame-level scores with human preference.

3. **CoT-style frame reasoning:** REACT applies chain-of-thought reasoning per sampled frame, producing both a point-wise quality score AND interpretable attribution labels for specific structural distortions. This makes evaluation results actionable — the system can say not just "this frame scores 3/10" but "this frame has texture corruption in the upper-left region."

**Concrete application to our system:**

- **Evaluation efficiency at L0-L2:** Our pipeline processes 120 frames/sec. Running expensive perceptual similarity (L3) or MLLM aesthetic evaluation (L4) on every frame is prohibitive. FMG-DFS provides a principled selection: compute cheap frame-to-frame pixel difference magnitudes (we already have this data from L0 delta computation), identify magnitude peaks (transitions, beat responses, major colour shifts), and only run expensive metrics on those frames. This could reduce L3/L4 evaluation cost by 80-90% while focusing on the frames that matter most.

- **Alignment with our existing L0 layer:** Our L0 already computes frame-to-frame absolute differences. These differences ARE the "flow magnitude" analogue for our 1D 320-pixel signal. Peaks in |Δframe| correspond to visual events. FMG-DFS formalises what we might do intuitively: run L3/L4 only on high-|Δ| frames.

- **GRPO training for frame-level scoring (future work):** If we ever train a lightweight frame-quality predictor (replacing or augmenting MLLM calls), the GRPO-based training with pairwise preference data is the method to use. This is particularly relevant for building an L3 surrogate that runs at inference time.

**Implementation complexity:** Low for the sampling method (reuse existing L0 delta magnitudes as the selection signal). Medium for the GRPO-trained frame scorer (requires preference dataset collection).

**Target layer:** Architecture (evaluation pipeline efficiency) + L0 (frame-level quality) + L3 (perceptual evaluation optimisation)

**Recency note:** Published January 7, 2026 — 61 days ago, borderline on the 60-day recency threshold. Included because the method is highly specific and directly applicable.

## Finding 2: Self-Play Evolution Requires Learnable Information Gain — Termination Criterion for Closed-Loop Optimisation

**Paper:** arXiv:2603.02218 — "Self-Play Only Evolves When Self-Synthetic Pipeline Ensures Learnable Information Gain" (February 10, 2026)
**Authors:** Wei Liu, Siya Qi, Yali Du, Yulan He

**One-sentence relevance:** Provides a formal criterion ("learnable information gain") for detecting when a self-improving optimisation loop has plateaued and should change strategy — directly applicable as a termination/adaptation signal for our PROPOSE→APPLY→VERIFY closed-loop optimiser.

**Key transferable method:**

1. **Learnable Information Gain (LIG) criterion:** Self-improving systems fail when the feedback loop produces data that doesn't contain novel learnable structure. The paper demonstrates that reward optimisation alone is "necessary but not sufficient" — a system may improve its reward score via reward hacking or memorised patterns while the actual learnable information content remains flat. LIG measures whether each iteration of the loop produces genuinely new learning signal.

2. **Proposer-Solver-Verifier (PSV) pattern with asymmetric co-evolution:** The three roles (proposal generation, solution execution, verification) operate on the same information source but along different synthetic directions. The key insight: all three roles must evolve TOGETHER. If the Proposer keeps generating similar proposals while the Verifier stays static, the loop stagnates.

3. **Three design principles for sustained evolution:**
   - **Asymmetric co-evolution:** Balance capability growth across roles so no single role becomes a bottleneck.
   - **Capacity growth:** Scale resources proportionally as the system improves.
   - **Proactive information seeking:** Inject external context to prevent the loop from becoming a closed echo chamber.

**Concrete application to our system:**

- **Optimisation loop stagnation detection:** Our PROPOSE→APPLY→VERIFY loop generates parameter modifications, applies them, and evaluates the result. If the proposer keeps suggesting similar modifications (e.g., always tweaking the same 2-3 parameters), the loop produces diminishing returns. LIG provides a principled metric: compute whether each cycle's evaluation data contains information the proposer hasn't already seen. If LIG drops below a threshold, the loop should: (a) expand the parameter search space, (b) change the proposal strategy, or (c) inject external diversity (random perturbation, different audio segments).

- **Reward hacking detection:** If our metric scores improve but the LED output is visually converging to a degenerate solution (e.g., always the same colour pattern regardless of audio), that's reward hacking. LIG measures whether the verifier is actually providing NEW quality signal vs. rubber-stamping familiar patterns. This is detectable: if the variance of evaluation scores across iterations drops while the absolute score plateaus, the loop has stagnated.

- **Asymmetric co-evolution for our system:** Our proposer (parameter modification engine) and verifier (metric pipeline) must evolve together. If we improve L0-L2 metrics but never update the proposal strategy, we'll converge to the ceiling of whatever the current strategy can achieve. The paper suggests periodically assessing whether the verifier's signal is still informative for the proposer's current capability level.

**Implementation complexity:** Medium — LIG as a concept requires defining "learnable information" in our domain. A practical proxy: track the information-theoretic entropy of proposed modifications across iterations. If proposal entropy drops, diversity is decreasing. If evaluation score variance drops while scores plateau, the verifier signal is stale.

**Target layer:** Optimisation (closed-loop evolution and termination)

## Finding 3: CAE-AV — Per-Frame Audio-Visual Agreement Gating for Weighted Evaluation

**Paper:** arXiv:2602.08309 — "CAE-AV: Improving Audio-Visual Learning via Cross-modal Interactive Enrichment" (February 9, 2026)
**Authors:** Yunzuo Hu, Wen Li, Jing Zhang

**One-sentence relevance:** Introduces a CASTE module that estimates per-frame audio-visual agreement scores and uses them to dynamically weight which frames contribute most to the learning/evaluation signal — directly transferable as a frame-importance weighting scheme for our L2 audio-visual alignment metric.

**Key transferable method:**

1. **Per-frame audio-visual agreement estimation:** CASTE evaluates frame-level audio-visual agreement by computing a consistency score at each timestep. Frames with high A-V agreement (audio event strongly corresponds to visual event) are upweighted; frames with low agreement (off-screen audio, background noise, no visual response) are downweighted.

2. **Dynamic spatial-temporal balancing:** The agreement score drives a dynamic balance between spatial attention (what's happening in the frame) and temporal attention (how the frame relates to neighbouring frames). High A-V agreement → emphasise spatial details. Low agreement → emphasise temporal continuity.

3. **CASE semantic guidance injection:** Uses a frozen CLIP encoder on generated captions to inject cross-modal semantic context, aligning features via attention for better spatio-temporal alignment assessment.

**Concrete application to our system:**

- **L2 weighted scoring:** Our current L2 audio-visual alignment metrics (BeatAlign, corr(|Δ|)) treat all temporal windows equally. But not all moments are equally important: beat drops, chord changes, and onset events are the moments where A-V alignment matters most. CAE-AV's per-frame agreement gating provides the pattern: compute expected A-V correspondence per frame (based on audio analysis output — beat probability, onset strength, spectral flux), then weight the L2 metric contribution of each frame by this expected correspondence. High-salience audio moments get higher weight in the final alignment score.

- **Bridging L0 and L2:** This creates a principled way to combine our L0 frame-level data with L2 alignment scoring. Instead of computing separate L0 and L2 scores, use the audio analysis to identify which frames SHOULD have high visual change, then check whether they DO.

**Implementation complexity:** Low — we already have per-frame audio salience data from the firmware's musical saliency analysis. Using it as a weighting signal for L2 metrics is straightforward.

**Target layer:** L2 (audio-visual alignment — frame importance weighting)
