## Finding 1: ToolRLA — Multiplicative Reward Decomposition for Multi-Dimensional Quality Scoring

**Paper:** arXiv:2603.01620 — "ToolRLA: Multiplicative Reward Decomposition for Tool-Integrated Agents" (March 2, 2026)
**Author:** Pengbo Liu

**One-sentence relevance:** Demonstrates that multiplicative (not additive) composition of multi-dimensional reward signals enforces strict ALL-dimensions-must-pass quality evaluation — directly applicable to composing our L0–L4 metric layers into a composite quality score.

**Key transferable method:**

1. **Multiplicative reward composition:** Instead of computing a weighted sum of sub-scores (additive: R = w₀·L0 + w₁·L1 + ... + w₄·L4), multiply normalised sub-scores: R = L0^α₀ × L1^α₁ × ... × L4^α₄. The critical property: if ANY sub-score approaches zero, the composite score collapses to zero regardless of other dimensions. This prevents the optimiser from producing outputs that excel on 3/5 layers but catastrophically fail on one.

2. **Ablation result — 7pp improvement over additive:** The paper demonstrates a 7 percentage point improvement in task completion when using multiplicative vs additive reward combination. The mechanism: additive rewards allow the optimiser to "trade off" dimensions (sacrifice tool selection accuracy for parameter accuracy). Multiplicative rewards make this trade-off structurally impossible — the geometric mean enforces balanced improvement across all dimensions.

3. **Exponent-weighted multiplicative composition:** Each dimension's exponent (αᵢ) controls relative importance without allowing total masking. α = 1.0 means standard weight; α > 1.0 amplifies that dimension's penalty for poor scores; α < 1.0 reduces sensitivity. This is strictly more principled than additive weighting because the zero-floor property is preserved at all weight settings.

**Concrete application to our system:**

- **Composite quality function redesign:** Our current approach (per the existing findings) contemplates additive composition with adaptive weights (EduVQA S2D-MoE pattern from 2026-03-09_2310.md) and regulariser-augmented MLLM scores (FlowPortrait pattern from 2026-03-09_2310.md). ToolRLA proposes a fundamentally different composition: multiplicative. The practical difference:
  - **Additive:** A configuration scoring L0=0.9, L1=0.1, L2=0.8, L3=0.7, L4=0.8 gets composite ≈ 0.66 (looks decent despite terrible temporal coherence).
  - **Multiplicative:** Same configuration gets composite ≈ 0.036 (correctly flags the L1 catastrophe).

- **Phase-appropriate composition switching:** Use multiplicative composition during exploration phases (reject configurations with ANY bad layer) and additive composition during exploitation phases (fine-tune trade-offs between layers). This gives a natural two-phase quality function: multiplicative as the gatekeeper, additive for final ranking among passed candidates.

- **Interaction with FlowPortrait's regulariser pattern:** FlowPortrait uses hard metrics as additive regularisers on MLLM scores. ToolRLA suggests an alternative: use hard metrics as multiplicative gates. Composite = MLLM_score × clamp(L1_temporal, min_threshold, 1.0) × clamp(L2_alignment, min_threshold, 1.0). The MLLM provides the primary quality gradient; the hard metrics enforce minimum thresholds via multiplication.

**Implementation complexity:** Low — changing from additive to multiplicative composition is a single-line code change. Calibrating exponents requires experimentation but can start with αᵢ = 1.0 (equal geometric weighting) and adjust.

**Target layer:** Architecture (metric composition strategy — how L0–L4 combine)

## Finding 2: Null-Space Projection for MLLM Judge Debiasing — and the Limits of Linear Debiasing

**Paper:** arXiv:2603.03291 — "One Bias After Another: Mechanistic Reward Shaping and Persistent Biases in Language Reward Models" (March 5, 2026)
**Authors:** Zhengxuan Wu, Daniel Yang, Atticus Geiger, Christopher Potts

**One-sentence relevance:** Provides a concrete activation-level debiasing technique (null-space projection) that removes length, position, and uncertainty biases from reward model outputs — directly applicable to improving our L4 MLLM-as-judge signal — AND demonstrates which biases resist linear intervention (sycophancy, model-style), establishing clear limits on what debiasing can achieve.

**Key transferable method:**

1. **Difference-of-means bias probe construction:** Identify a bias direction in the model's hidden state space by computing the difference of mean activations between positive examples (e.g., short-but-correct evaluations) and negative examples (e.g., long-but-wrong evaluations). This produces a directional vector p = normalize(mean(h₊) − mean(h₋)) that encodes the bias.

2. **Null-space projection for debiasing:** Remove the bias direction from activations at inference time: h_debiased = h − α(pᵀh)p, where α controls projection strength. This surgically removes the bias signal without retraining the model. Multiple bias directions can be removed simultaneously by projecting out a subspace.

3. **Three biases successfully removed (low-complexity):**
   - **Length bias:** MLLM judges prefer verbose evaluations regardless of correctness. Null-space projection removes this, so a concise evaluation "beat alignment is poor" scores equivalently to a verbose one.
   - **Uncertainty bias:** MLLM judges over-penalise hedged assessments ("the temporal coherence seems acceptable"). Removing this improves calibration for ambiguous cases.
   - **Position bias:** MLLM judges systematically prefer certain positions in pairwise comparisons (e.g., always preferring the first option presented). Removing this is critical for pairwise ranking in our evaluation pipeline.

4. **Two biases that PERSIST (high-complexity, critical warning):**
   - **Sycophancy:** Agreement rates with incorrect user prompts exceed 24% even in SOTA models. If our MLLM judge is given evaluation context that hints at what the "right answer" should be, it will agree with that hint even when the LED output is objectively poor. **Implication: do NOT include hints about expected quality in L4 evaluation prompts.** This is a structural constraint on prompt design.
   - **Model-style sensitivity:** Reward models correlate with specific writing styles, explaining 4–16% of score variance. For our MLLM judge, this means the STYLE of the evaluation prompt influences scores beyond the CONTENT. **Implication: CyclicJudge rotation (from 2026-03-09_2341.md) helps, but style sensitivity is an irreducible noise floor.**

**Concrete application to our system:**

- **L4 debiasing protocol:** Before deploying an MLLM judge in the optimisation loop:
  1. Generate paired evaluation inputs that differ ONLY in length/position/hedging.
  2. Compute bias probes from the activations (requires access to model internals — works with open-weight models like Llama, not with API-only models like GPT-4).
  3. Apply null-space projection at inference time.
  4. Verify that debiased scores no longer correlate with the bias dimension.

- **API-model workaround:** For closed-source MLLM judges where we can't access activations, the FINDINGS about bias structure are still actionable:
  - Control for length by normalising evaluation output length (reject evaluations shorter than X or longer than Y characters).
  - Control for position by randomising comparison order (already addressed by CyclicJudge).
  - Control for sycophancy by removing quality hints from evaluation prompts.

- **Noise floor estimation:** The paper establishes that model-style sensitivity explains 4–16% of reward variance. This gives us a concrete noise floor for MLLM-based evaluation: even with CyclicJudge rotation AND null-space debiasing, ~4–16% of L4 score variance is irreducible style noise. This means the optimiser cannot reliably distinguish quality differences smaller than this noise floor — setting a minimum detectable effect size for L4-guided optimisation.

**Implementation complexity:** Medium for open-weight models (requires activation extraction and probe computation — one-time setup). Low for API models (use the bias-awareness findings to design better prompts and evaluation protocols). The noise floor estimation requires a calibration dataset.

**Target layer:** L4 (aesthetic evaluation — judge debiasing and noise floor characterisation)

**Differentiation from existing findings:**
- CyclicJudge (2026-03-09_2341.md) handles *allocation* — cycling prompt variants to cancel systematic bias. Null-space projection handles *activation-level debiasing* — removing bias from the model's internal representations.
- M-JudgeBench (2026-03-09_2341.md) handles *capability assessment* — testing whether the judge can rank correctly. This paper handles *bias identification and removal* — a different intervention.
- Composition: M-JudgeBench validates capability → null-space projection removes known biases → CyclicJudge rotates prompt variants → FlowPortrait structures the composite reward. This builds toward a complete L4 quality pipeline.
