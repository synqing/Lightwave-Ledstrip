DESIGN DECISIONS
3 SCREENS ONLY
Minimalist onboarding. Users want to control lights, not read tutorials. Each screen has one clear purpose: introduce, connect, demonstrate responsiveness. The "I've used this before" escape hatch on screen 1 respects returning users.
Total flow: ~20 seconds for new users
WIFI GUIDANCE
K1 operates as an AP (access point), not a router. This is non-obvious. Screen 2 explains the topology visually: K1 broadcasts, phone connects. The "Open WiFi Settings" button uses iOS deep-link to reduce friction. No password required (open network) eliminates another failure point.
AP: LightwaveOS · No password · 192.168.4.1
IMMEDIATE FEEDBACK
Screen 3 lets users tap an effect and see it change instantly on the physical K1. This demonstrates the core value proposition: the app controls real lights in real time. The LED preview strip updates simultaneously, reinforcing the connection between screen and hardware.
Effect change latency: <50ms (REST API)
VISUAL CONTINUITY
The animated LED strip on the welcome screen uses the same centre-origin rendering as the physical device. Users see the lighting pattern before they even connect, building anticipation. The gold-to-cyan gradient previews the brand palette.
Animation: 60fps CSS, centre origin 40/41 (of 80)
ACCESSIBILITY
All text meets WCAG AA contrast (4.5:1 minimum on #0F1219 backgrounds). Touch targets are 52pt (buttons) and 44pt minimum (effect cards). Progress dots provide orientation for screen reader users. Motion-sensitive users: animations respect prefers-reduced-motion.
Contrast: Primary text 12.8:1 · Gold 8.5:1
TYPOGRAPHY HIERARCHY
Welcome screen uses full brand stack: Bebas Neue for "LIGHTWAVEOS" title, Rajdhani for body copy, JetBrains Mono for the WiFi address. Step indicators ("STEP 2 OF 3") use uppercase Bebas at small size for navigational context without visual weight.
Title: 36pt Bebas · Body: 15pt Rajdhani 500
