Zones Tab — Redesign Notes

Visual Strip Diagram

72pt LED strip canvas shows all 320 LEDs live (two rows, 160 each). Each zone gets its own distinct colour region as defined by lwZone0–3.
Draggable boundary handles — glowing vertical lines on the strip. Drag left/right to adjust zone split points. Matches existing ZoneSplitSlider semantics but more visual and direct.
Centre origin marker (79|80) always visible. Helps users understand the physical LED layout.

Zone Count Selector

1–4 zone pills — selecting instantly updates the strip diagram and zone card list. 1 = full-strip mode (no zones), 2 = inner/outer, 3 = inner/middle/outer, 4 = maximum segmentation.
Zones On toggle — when off, all zone configuration is preserved but the K1 runs the global effect. Clear state communication.

Zone Card Design

Pill title overlapping card top border — existing pattern from ZonePillTitle. Each zone uses its canonical colour: Zone 1 = Cyan, Zone 2 = Green, Zone 3 = Purple, Zone 4 = Orange.
Effect/palette rows use ◀/▶ nav buttons matching existing ZoneCard.swift. 46pt tall rows, well within thumb reach.
Speed and brightness as inline mini-sliders within each zone card. No secondary cards needed.
Per-zone enable toggle in card header — matches existing ZoneConfig.enabled field. Disabling greys out the card content.

Zone Colour System

Zone 1 — Inner (lwZone0 Cyan)
Zone 2 — Middle (lwZone1 Green)
Zone 3 — Outer (lwZone2 Purple)
Zone 4 — Outermost (lwZone3 Orange)

Motion Design Intent

Boundary handle drag: haptic at each 5-LED increment (selectionChanged). Strip diagram updates at 30fps during drag.
Zone count change: spring animation — old zone cards slide out (scale+fade), new cards slide in (staggered 40ms per card).
Zone enable toggle: strip canvas region dims (opacity 0.15) for disabled zone. Immediate visual feedback without modal.

Architecture Alignment

Zone split maps to boundary0 / boundary1 in AppViewModel.zones. LED range is 1–78 per existing ZoneSplitSlider.
Strip diagram renders ZoneSegment.s1LeftStart / s1LeftEnd visually. Centre origin at 79/80 is always respected — no linear sweeps in visual representation.
