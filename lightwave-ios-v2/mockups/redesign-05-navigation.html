Navigation Notes

Tab Bar Design Principles

44pt icon wrap + 28pt icon — meets WCAG 2.5.5. The icon-wrap background provides visual feedback on tap without requiring precise pixel targeting.
Bottom indicator pill — 20×3pt gold pill below active tab. Subtle, not competing with content. Uses the same lwGold as all primary accents.
Audio tab beat badge — pulsing 7pt dot on the Audio tab icon when audio is actively detecting beats. Visible at a glance without needing to switch to Audio tab.
82pt tab bar height — respects safe area. 10pt top padding before icons so thumb lands comfortably. Bottom gap for home indicator.

Persistent Status Bar

Connection dot (7pt) + brand name always top-left. Users can always identify the connected device without navigating to Device tab.
IP address top-right in JetBrains Mono. When disconnected, replaced by "OFFLINE" in red.
Status bar content is tab-contextual: Play shows "LightwaveOS", Audio shows "ESV11 • LIVE", Zones shows zone count, Device shows firmware version.

Connection Sheet

Auto-presents on first launch or when connection drops for >3 seconds. Drag-to-dismiss when reconnected.
K1 AP (LightwaveOS_K1 at 192.168.4.1) listed prominently. One-tap connect. No SSID/password entry — K1 is open network by design.
Sheet is non-blocking — user can still see app content behind it. Controls are disabled with visual overlay but content is visible.

Tab Transition Animation

Horizontal slide between tabs — slide direction matches visual order (left→right = Play→Zones→Audio→Device). Consistent with iOS navigation metaphor.
Spring: damping 0.85, stiffness 300, mass 0.8. Fast, snappy, not floaty. Duration ~200ms.
Tab bar icons animate: previous icon scale(1.0 → 0.85), next icon scale(0.85 → 1.0). Reinforces directional slide.
