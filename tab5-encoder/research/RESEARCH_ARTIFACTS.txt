KEY FINDINGS (TL;DR)
================================================================================

ROOT CAUSE:
• Master resets mid-I2C transaction while slave pulls SDA LOW
• Slave firmware crash while SDA pulled → stuck LOW forever
• STM32F030 has NO I2C timeout (hardware limitation)
• Errata 2.9.5: Slave needs 50-100µs delay after STOPF

RECOVERY METHODS (in order of likelihood):
1. Wait 50ms (clock stretching may recover on its own) — 10% success
2. SCL pulse recovery (9 clock cycles) — 50% success
3. Master I2C reset — 20% success
4. Slave power cycle (Grove power toggle) — 99% success

CONCLUSION:
SDA stuck LOW is a known hardware limitation requiring multi-layered recovery.
Power cycle is the only reliable solution (99% success rate).

HARDWARE SPECIFICATIONS
================================================================================

M5ROTATE8:
├─ Microcontroller: STM32F030F4/F6/F8
├─ I2C Address: 0x41 (default, reprogrammable via 0xFF)
├─ Power: 5V input → internal 3.3V DC-DC
├─ I2C Voltage: 3.3V (STM32F030 I/O standard)
├─ Pull-ups: 3.3V external (Grove connector)
├─ Firmware: Open source (github.com/m5stack/M5Unit-8Encoder-Internal-FW)
└─ Boot Time: ~2ms (reset) + 50ms (init), total 500ms for power cycle recovery

STM32F030 I2C Limitations:
├─ NO hardware I2C timeout
├─ Open-drain output (can only pull LOW, not drive HIGH)
├─ Errata 2.9.5: Slave mode timing constraint
├─ Requires 50-100µs delay between STOPF and re-enabling listener
└─ No built-in bus recovery (relies on master timeout)

M5Stack Grove Connector:
├─ SDA: GPIO21 (ESP32 master)
├─ SCL: GPIO22 (ESP32 master)
├─ Power: 5V from battery/USB
├─ GND: Common reference
└─ No hardware reset pin accessible from Grove

RECOVERY DECISION TREE
================================================================================

Is SDA stuck LOW? (GPIO21 read)
 ├─ YES: [PROBLEM DETECTED]
 │   ├─ Wait 100ms for natural recovery
 │   │   └─ SDA released? → SUCCESS (probably clock stretching)
 │   │   └─ SDA still LOW? → Continue
 │   │
 │   ├─ Apply 9 SCL pulses (GPIO22 bit-bang)
 │   │   └─ SDA released? → SUCCESS (slave firmware responded)
 │   │   └─ SDA still LOW? → Continue (slave firmware hung)
 │   │
 │   ├─ Reset master I2C (Wire.end() + Wire.begin())
 │   │   └─ Ping slave? → SUCCESS (master was confused)
 │   │   └─ No ACK? → Continue
 │   │
 │   └─ Power cycle slave (Grove 5V toggle)
 │       └─ Slave responds? → SUCCESS (99% success rate)
 │       └─ Still hung? → HARDWARE FAILURE (replace M5ROTATE8)
 │
 └─ NO: [BUS IDLE] — No action needed
