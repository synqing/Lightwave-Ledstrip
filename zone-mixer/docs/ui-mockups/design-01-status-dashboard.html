Zone Mixer -- Design 01: Status Dashboard

AtomS3 0.85" LCD (128x128) UI Mockup

Design Rationale

The Status Dashboard prioritises information density over aesthetics. On a 0.85" 128x128 screen, every pixel matters. The layout uses a strict vertical stack with horizontal key-value pairs, maximising the number of state indicators visible at a glance.

Layout Strategy

  • Top bar (12px): WiFi and WebSocket connection indicators -- the two things most likely to need immediate attention. Colour-coded icons (filled/hollow circles) give instant status without reading text.
  • Master section (22px): Brightness and speed as horizontal bars with numeric readout. Bars use the full width for maximum resolution on a tiny screen.
  • Zone section (54px): Three zone rows, each showing zone number, enabled state, brightness bar, and brightness value. Zone colours match the physical LED colours on the Scroll wheels and 8Encoder units (cyan/green/orange).
  • Effect name (14px): Current effect name, truncated with ellipsis if too long. This is the most variable-length field, so it sits at the bottom where truncation is least disruptive.
  • Bottom status (10px): Zone count and zone system enabled/disabled.

M5GFX Implementation Notes

  • All rendering uses fillRect(), drawRect(), and drawString() -- no images or sprites needed.
  • Font: M5GFX built-in monospace fonts at size 1 (8px height) with setFont(&fonts::Font0) or setFont(&fonts::TomThumb) for the smallest text.
  • Colour constants map directly to M5GFX 16-bit colour values.
  • Full redraw at 2-4 Hz is sufficient -- state changes are infrequent.
