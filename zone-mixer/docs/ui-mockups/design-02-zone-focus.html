Design Rationale

Concept

Zone Focus dedicates the entire 128x128 screen to the three zone brightness levels. The bars are the primary visual element -- thick, colour-coded, and readable at arm's length while hands are on the scroll wheels. This is a "glanceable instrument" design: the user looks down, sees three bars at different widths, and knows exactly where each zone sits without reading numbers.

Colour System

  • #00FFFF Zone 1 (cyan) -- matches Scroll CH0 LED
  • #00FF99 Zone 2 (green) -- matches Scroll CH1 LED
  • #FF6600 Zone 3 (orange) -- matches Scroll CH2 LED
  • #FFFFFF Master (white) -- in footer only
  • Bar backgrounds use a dark-tinted version of each zone colour for depth contrast

M5GFX Implementation Notes

  • All rendering uses fillRect() and drawString() -- no images, no sprites
  • Bar width = (brightness * 124) / 255 (124px usable width with 2px margins each side)
  • Font: use M5GFX built-in font at size 1 (8px) for status/footer, size 2 (16px) for zone labels and values
  • Redraw only changed bars on each frame -- clear and refill the specific zone row
  • Text shadow effect: draw black text at +1,+1 offset, then white text on top
  • Status dots: fillCircle() with 2px radius

Readability at 0.85"

  • 30px bars at 0.85" physical size are approximately 5mm tall -- large enough for peripheral vision
  • Zone labels ("Z1", "Z2", "Z3") are 11px -- legible at 40cm viewing distance
  • Brightness values (12px) are the largest text on screen after the bars themselves
  • The bar width itself is the primary information channel -- numbers are secondary confirmation
  • High contrast: white text on saturated colour bars with dark text shadow for readability on both filled and unfilled regions

State Variations

  • Disconnected: Status dots turn red, bars dim to 30% opacity
  • Connecting: WS dot turns blue, pulses
  • Zone disabled: Bar fill turns grey (#333333), value shows "--"
  • Zero brightness: Bar fill width = 0, value shows "0", bar background still visible
  • Max brightness: Bar fills entire row, value shows "255"

Design Trade-offs

  • No effect names: these are available on the K1 display and iOS app. Screen space is better used for what only the mixer can show -- per-zone fader state
  • No master bar: master brightness is shown as a small number in the footer. The master scroll wheel has its own LED for feedback; duplicating it as a fourth bar would shrink the zone bars
  • No encoder mode indicators: the 8Encoder units have their own LED rings that show mode state in colour. Duplicating that information here wastes the screen's limited real estate
