Design 04 -- Minimal Glance

Zone Mixer UI for AtomS3 (128x128 LCD) -- ultra-minimal, change-focused

4x Scale (512 x 512)
1:1 Actual (128 x 128)

Design Rationale

Philosophy

The Minimal Glance design treats the 0.85" screen as a peripheral indicator, not a primary interface. The user's hands are on the encoders; the screen confirms what changed. One glance -- one number -- done.

Why This Works at 128x128

  • 5 elements maximum -- WiFi dot, WS dot, master brightness, effect name, zone dots. Nothing else competes for attention.
  • 48px master number fills ~37% of vertical space. Readable at arm's length even on a 0.85" panel.
  • Corner indicators use 6px dots -- small enough to ignore, visible enough to notice a red flash.
  • Zone dots use proportional diameter (4--10px) to encode brightness without needing numbers. Three dots for three zones, colour-coded to match physical encoder rings.

Colour Choices

  • Master brightness: pure white #FFFFFF -- maximum contrast on black, no colour association.
  • WiFi dot: green (connected) / red (disconnected).
  • WS dot: green (open) / yellow (connecting) / red (closed).
  • Zone 1: cyan #22d3ee -- cool, distinct from green status dots.
  • Zone 2: green #4ade80 -- natural, mid-spectrum.
  • Zone 3: orange #fb923c -- warm, high contrast against black.
  • Effect name: dim grey #666666 -- readable when sought, invisible when not.

Performance Considerations

  • Only 5 draw calls per frame -- trivial for the AtomS3 ST7789 driver.~0.3ms
  • No gradients, no anti-aliasing, no alpha blending needed. Pure filled primitives.
  • Full-screen clear is black -- can use hardware fill if available.
  • Partial updates viable: only redraw the master number on encoder turn, dots on state change.

Accessibility at Scale

  • Master number contrast ratio on black: 21:1 (exceeds WCAG AAA).
  • Status dots use position (top-left vs top-right) as secondary encoding -- not colour alone.
  • Zone dots use both colour and size -- dual encoding for colour-vision deficiency.
WiFi Indicator (top-left) Position: (4, 4). Radius: 3px. Green when AP connected, red when disconnected. Stays peripheral -- users rarely need this after initial setup.
WebSocket Indicator (top-right) Position: (123, 4). Radius: 3px. Green/yellow/red tri-state. Yellow (connecting) is critical -- tells user to wait, not retry.
Master Brightness (centre) Font: 48px, bold, white. Vertically centred at y=52 (baseline). Horizontally centred via text width calculation. Displays 0--100.
Zone Brightness Dots (bottom) Y: 114. Spaced at x=42, x=64, x=86 (22px gap). Diameter scales 4--10px proportional to zone brightness (0--100%). Colour-coded per zone.
