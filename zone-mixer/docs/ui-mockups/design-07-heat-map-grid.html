Design Rationale

Concept

The Heat Map Grid eliminates text entirely. Every parameter is a coloured cell whose brightness and saturation encode its current value. At a glance from 1 metre away, the operator reads the full system state as a colour pattern — no parsing, no squinting at 6pt labels on a 0.85" screen.

Why This Works at 128x128

  • Text is illegible below ~8pt on a 0.85" LCD at arm's length. This design uses zero text — only colour fields.
  • Large cells (38x38 px for zones, full-width bars for master/speed) are visible in peripheral vision during performance.
  • The 2px gap grid creates a clean separation without wasting space on borders or dividers.

Colour Mapping

  • Zone 1 — Cyan: #001a1a (off) to #00FFFF (full)
  • Zone 2 — Green: #001a0d (off) to #00FF66 (full)
  • Zone 3 — Orange: #1a0e00 (off) to #FF8800 (full)
  • Master — White: #1a1a1a (off) to #FFFFFF (full)
  • Speed — Blue: #000d1a (off) to #4488FF (full)

Status Indicators

  • WiFi: bright green = connected, red = disconnected. Binary state, no ambiguity.
  • WebSocket: green = open, amber = connecting/retrying, red = closed. Three-state traffic light.
  • Zone enabled: green = active, near-black = disabled. Mirrors the top row — disabled zones are dark above AND below, reinforcing the visual pattern.

Grid Layout (128x128 px)

All coordinates in native pixels. Gap = 2px throughout.

  • Row 1 (y: 2..40): Three 38x38 zone brightness squares, left-to-right, 2px gaps
  • Row 2 (y: 42..72): Master brightness bar (80x30) + speed bar (38x30), 2px gap
  • Row 3 (y: 74..96): Three 38x22 status cells — WiFi, WebSocket, zone-enabled indicator

Implementation Notes for LVGL / Direct Framebuffer

  • Zero font rendering — pure lv_canvas_set_px or fillRect() calls. No font ROM needed.
  • Colour interpolation: lerp(dark, vivid, value/255.0) per channel. One multiply per channel per cell.
  • Update only changed cells — dirty-flag each parameter, redraw only modified rectangles. Worst case: 8 cells = 8 fillRect calls.
  • Entire render fits in <0.5ms on ESP32-S3 at 240 MHz.

Limitations

  • No numeric readout — operator cannot tell "brightness = 147" vs "153". Acceptable: zone mixing is feel-based, not precision.
  • Colour-blind accessibility: cyan/green/orange chosen for deuteranopia separation, but full accessibility requires user testing.
  • Three zones maximum — adding a fourth zone would require smaller cells or a second row, breaking the at-a-glance scan.
