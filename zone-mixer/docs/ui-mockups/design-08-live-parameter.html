Design Rationale

Why "Live Parameter"

The operator's primary interaction with the Zone Mixer is turning encoders. On a 0.85" screen, trying to show all parameters at all times forces everything to be unreadably small. The Live Parameter pattern solves this by making the screen reactive to the operator's hands — the last thing you touched is the thing you want to see.

This mirrors DJ hardware (Pioneer CDJ, Allen & Heath Xone) where turning a knob temporarily displays its value on the nearest screen. The operator never needs to search for the right readout — it appears automatically where their attention already is.

48px Value — Readability at Arm's Length

At 128px screen height, a 48px value occupies 37.5% of the vertical space. On the physical 0.85" (21.6mm) screen, that renders each digit at roughly 8mm tall — legible from 60cm away in dim lighting. This is critical: the operator is looking at the LED art, not at the controller. A quick glance downward must be sufficient.

Zone Colour Coding

The 8px colour strip at the top of the Active state provides instant zone identification through colour alone — no reading required. Blue (Zone 1), amber (Zone 2), red (Zone 3). These colours are consistent across the entire Zone Mixer system: encoder LEDs, screen UI, and the K1 dashboard.

Previous-to-New Delta

The struck-through previous value alongside the new value answers "how much did I just change it?" without requiring the operator to remember the starting value. This is especially useful during performance where small adjustments matter and the operator needs confirmation that the change registered.

2-Second Timeout

Two seconds is the sweet spot: long enough to read the value after releasing the encoder, short enough to avoid stale information. If the operator continues turning, the timeout resets continuously — the Active state persists as long as there is input. After 2 seconds of stillness, the screen gently returns to the Idle summary.

Accessibility on Embedded Hardware

WCAG contrast ratios are not directly applicable to emissive OLED/LCD at these sizes, but the design follows the principle: white (#FFF) on black (#000) for the value gives maximum contrast. Colour is never the sole information carrier — the parameter name text always accompanies the zone colour strip. The zone dots provide positional redundancy for the colour coding.

Implementation Notes

On the AtomS3 (ST7789 driver, 128x128, SPI), this design requires only two font sizes (10px and 48px), three zone colours, and simple rectangle fills. No alpha blending, no gradients, no anti-aliasing needed. The transition from Idle to Active is a full-screen redraw — no animation required on the embedded side. The 2-second timeout is a simple millis() check in the display update loop.
