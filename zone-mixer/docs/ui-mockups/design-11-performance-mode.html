Design Rationale — Performance Mode

Philosophy

Designed for performers operating in near-darkness. Every design choice prioritises instant readability at arm's length on a 0.85" screen. Minimal brightness output to avoid polluting the light show with screen glare. The display should be a tool, not a light source.

Page 1: LIVE — DJ Fader Bars

  • Three vertical bars arranged like DJ channel faders — the most instinctive physical metaphor for the encoders the user is turning.
  • Bar fill is proportional (0-255 range). Zone colours are chosen for maximum mutual contrast even under colour vision deficiency: #00E5FF cyan (Z1), #00FF88 green (Z2), #FF8800 orange (Z3).
  • Master brightness as a thin 4px white bar at right edge — always visible but subordinate to per-zone bars. Provides global context without consuming valuable horizontal space.
  • Only text: single-digit zone IDs ("1" "2" "3" "M") at the bottom. No numeric values — the bar height IS the information.
  • A small page indicator dot row at bottom-centre shows which carousel page is active.

Page 2: EFFECT — Current Effect Info

  • Effect name rendered at 14px equivalent in amber (#FFB000) — amber has highest visibility on pure black in dark-adapted vision.
  • Speed bar below: thin horizontal bar with #4488FF fill against dark track. Fills proportionally (0-255).
  • EdgeMixer mode at bottom in dim grey — reference information, not primary. Performers rarely change mode mid-show.

Page 3: STATUS — Connection Health

  • WiFi RSSI shown as 4 signal strength bars — universally understood without text. Bars light up green proportionally to signal strength.
  • WebSocket state: green dot = connected, red dot = disconnected. Binary state needs only binary encoding.
  • IP address in dim monospace — only needed for troubleshooting, kept small to not dominate.
  • Zone count and enabled/disabled state: green "3 ZONES ON" or red "ZONES OFF" — immediately communicates operational status.

Live Parameter Overlay

  • Full-screen takeover on any encoder turn. Black background with zero visual noise. The parameter becomes the ONLY thing on screen.
  • Parameter name at top in zone-coloured 10px text — identifies which zone or parameter was changed.
  • 48px value centre-screen in white — readable at 3m distance on the 0.85" display. This is the reason the overlay exists: large enough to read without bringing the device close during a live performance.
  • Thin colour bar at bottom shows position within the 0-255 range as a proportional fill. Provides spatial context ("am I near the top?") without requiring the performer to interpret numbers.
  • Auto-fades back to current page after 2 seconds. No button press needed.

Colour System — Dark Venue Optimised

  • #000000 — True black background. On LCD, this is the lowest emission. Prevents screen glow in dark venues.
  • #00E5FF — Zone 1 cyan. High blue content reads well in scotopic (dark-adapted) vision.
  • #00FF88 — Zone 2 green. Peak human spectral sensitivity. Maximum visibility at minimal brightness.
  • #FF8800 — Zone 3 orange. Warm contrast against cyan/green. Distinguishable even with deuteranopia.
  • #FFB000 — Amber accent for effect names. High visibility, low cognitive alarm (unlike red).
  • #FFFFFF — Master/value text. Reserved for the most important numeric readout.

Typography Constraints

At 128px width, all text is rendered via a 5×7 bitmap pixel font at integer coordinates. No sub-pixel rendering, no anti-aliasing — this matches the AtomS3 LCD's native pixel grid. The "48px" overlay value uses an 8×12 large bitmap font for maximum readability. At 4× scale, each logical pixel becomes a 4×4 CSS pixel block.

Accessibility

  • All active colours exceed 7:1 contrast ratio against #000.
  • Zone colours form a cyan/green/orange triad — distinguishable under protanopia, deuteranopia, and tritanopia.
  • Bars provide spatial encoding redundant with numeric values (on overlay).
  • Page indicators use size + brightness to indicate active state.

LVGL Implementation Notes

Page 1 bars map to lv_bar objects oriented vertically. The overlay triggers via lv_timer_create with 2000ms single-shot. The carousel uses lv_obj_set_scroll_snap_x on a parent container with button-press advancing lv_obj_scroll_to_x. The large overlay font would be a custom lv_font_t bitmap at 48px or nearest available.
