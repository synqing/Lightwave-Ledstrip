Design Rationale

Philosophy

Two pages cover 95% of live performance needs. The MIX page is a complete cockpit — zone brightness, master, and speed are all visible at once. The INFO page is a diagnostic glance checked between sets. Fewer pages means less button fumbling in dark venues where every mispress costs attention.

Colour System

  • #FFB000 — Amber accent. Page titles, effect name, active indicators.
  • #FF4444 — Zone 1. Warm red, strongest visual weight.
  • #44FF44 — Zone 2. Green, maximum contrast against Zone 1.
  • #4488FF — Zone 3 / Speed. Cool blue, completes the RGB triad.
  • #FFFFFF — Master bar. White distinguishes global from per-zone.
  • #333 — Bar track backgrounds. Visible without competing with fills.
  • #22CC44 — Connected / enabled status. Muted green to avoid distraction.
  • #CC2222 — Disconnected / disabled status.

MIX Page Layout

  • y=0-12 Header: Page title "MIX" in amber with speed value in top-right corner. Dual purpose: header + always-visible speed readout.
  • y=14-15 Separator: 1px line divides header from content.
  • y=18-95 Zone Bars: Three zone rows at 26px each. Zone number, full-width bar, and numeric value. Generous vertical spacing improves glanceability under stage lighting.
  • y=97 Separator: Divides zones from master.
  • y=100-118 Master: White bar spans full width with "M" label. Visually distinct from coloured zone bars. Larger than per-zone bars to reflect hierarchical importance.
  • y=122-127 Page Dots: Two dots centred at bottom. Active page = white, inactive = #333. Minimal navigation affordance.

INFO Page Layout

  • y=0-12 Header: Page title "INFO" in amber. Immediately identifies the diagnostic page.
  • y=18-34 Connection: WiFi and WS status with coloured dots and text labels. Binary state is instantly readable.
  • y=42-58 Effect: Current effect name in amber on two rows if needed. This is the primary diagnostic: "what is playing?"
  • y=66-82 EdgeMixer: Mode label with current state. Shows whether EdgeMixer is BLEND, EDGE, or OFF.
  • y=90-106 Zones: Zone count and enabled/disabled state. A quick confirmation that the correct zone configuration is active.
  • y=122-127 Page Dots: Same two-dot indicator, second dot active.

Overlay Behaviour

When any encoder is turned, the overlay appears immediately on top of whatever page is showing. It displays: the parameter name (e.g. "ZONE 1 BRIGHT"), the new value at 48px height for visibility across the stage, and a zone-coloured accent bar at the bottom. After 2 seconds of no input, the overlay fades and the previous page resumes. The overlay uses a semi-transparent dark background (#0a0a0a at ~90% opacity simulated) so the operator retains a faint sense of the underlying page.

Accessibility on Hardware

All text exceeds 7:1 contrast ratio against #0a0a0a. Zone colours use the red/green/blue triad which remains distinguishable under common colour vision deficiencies. Numeric values alongside bars provide redundant encoding. The two-page design itself is an accessibility feature: fewer pages means fewer chances to get lost in the interface under pressure.
