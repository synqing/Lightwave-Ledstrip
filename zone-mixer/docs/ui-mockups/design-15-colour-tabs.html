Design Rationale

Why Colour-Coded Tabs

On a 0.85" screen, text labels for page identity are too small to read at performance distance. A 4px colour bar at the top is visible from over a metre away and builds instant muscle memory: cyan means zones, amber means effects, violet means EdgeMixer, green means connection. The operator stops reading and starts recognising.

This pattern is borrowed from pro audio mixers and DAW channel strips where colour coding is the primary identification mechanism, with text serving only as confirmation.

Accent Colour Selection

  • #00BCD4 Cyan — Page 1 (Zone Overview). Cool, neutral, dashboard feel. Distinct from zone colours.
  • #FFB300 Amber — Page 2 (Effect & Speed). Warm attention colour, matches existing effect name styling.
  • #AB47BC Violet — Page 3 (EdgeMixer). Unique hue not used elsewhere, immediately distinguishable.
  • #4CAF50 Green — Page 4 (Connection). Green = healthy connection, semantic match to page content.

These four hues are maximally separated on the colour wheel and remain distinguishable under protanopia, deuteranopia, and tritanopia. No two accents share a confusion pair.

Page Layout Architecture

Each 128x128 page is divided into three vertical zones: the 4px accent bar (identity), 120px content area (information), and 4px dot row (navigation). The content area is further structured per page type, but all pages share the same vertical rhythm with consistent margins (4px sides, 2px internal gaps).

Region Height Purpose
Accent bar 4px Page identity — colour alone, no text needed
Content area 120px Page-specific information
Navigation dots 4px Page position — active dot matches accent colour

Live Parameter Overlay

When an encoder is turned, the overlay takes over the full screen with the parameter name, a 48px value (readable at 60cm), and a progress bar in the current page's accent colour. After 2 seconds of inactivity, the overlay fades and the current page returns. The overlay does not change which page is active — it layers on top temporarily.

The accent-coloured bar in the overlay reinforces which page context the parameter belongs to, maintaining the colour-coded identity system even during transient feedback.

Zone Colour System

  • #FF4444 Zone 1 — warm red, highest visual priority
  • #44FF44 Zone 2 — green, maximum separation from Zone 1
  • #4488FF Zone 3 — blue, completes the RGB triad
  • #FFFFFF Master — white, distinct authority over all zones

Implementation Notes for AtomS3

  • ST7789 driver, 128x128, SPI interface. Full-screen redraws are fast (<10ms).
  • Two font sizes needed: 5x7 bitmap (labels, values) and a large ~48px numeral set (overlay value).
  • Page state is a single uint8_t pageIndex with modulo-4 cycling on button press.
  • Overlay is a boolean flag + millis() timestamp. If millis() - overlayStart < 2000, render overlay instead of page.
  • Accent colours stored as a 4-element uint16_t[] in RGB565 format.
  • Navigation dots: four 3px filled circles at y=124, spaced evenly, active dot uses accent colour.

Accessibility

All text exceeds WCAG AA contrast on the #0a0a0a background. Zone identification uses both colour and position/label — never colour alone. The 4px accent bar is supplemented by page dots at the bottom (positional encoding) and distinct content layouts (structural encoding). Signal strength uses bar height progression alongside colour.
