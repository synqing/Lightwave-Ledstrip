Design Rationale

Concept

A three-page carousel with smooth horizontal slide transitions between pages. Each page serves a distinct function: MIX (DJ fader bars for zone brightness), FX (current effect name with speed arc gauge), and NET (connection status with animated signal strength). Transitions provide spatial context — the user understands pages exist "next to" each other, not as disconnected states.

Transition System

  • Page slide: Old page slides left, new page slides in from right. Duration: ~200ms (6 frames at 30fps). At 50% both pages are visible side-by-side, each occupying half the 128px width. On M5GFX this is two fillRect redraws per frame with X-offset animation.
  • Live overlay: A parameter value panel slides up from the bottom (32px tall). Appears when any encoder changes a value. Auto-hides after 2 seconds by sliding back down. On M5GFX: single fillRect with Y-offset.
  • Disconnect flash: Red 2px border pulses 3 times (1 frame red, 1 frame normal) when WebSocket connection drops. Visual alarm without disrupting page content. On M5GFX: drawRect in red, then black, 3 cycles.

Visual Language

  • Zone 1 — Cyan (#00E5FF)
  • Zone 2 — Green (#00E676)
  • Zone 3 — Orange (#FF9100)
  • Master — White (#E0E0E0)
  • FX Accent — Amber (#FFB833)
  • NET Accent — Sky (#33CCFF)

Page 1: MIX (DJ Fader Bars)

  • Four vertical fader tracks: Z1, Z2, Z3, Master (same layout as Design 05).
  • Gradient fills, 2px cap lines, numeric values inside tracks.
  • Master track is 30px wide (vs 24px zone tracks) for hierarchy.
  • Status icons (WiFi/WS) at top-left, speed readout at bottom-centre.

Page 2: FX (Effect Display)

  • Effect name centred in large text (13px), amber accent colour.
  • Speed arc gauge: 180-degree arc (bottom half), filled proportionally to speed value.
  • Zone count shown below the arc: "3 ZONES" with dim indicator dots.
  • Page dot indicators at bottom (3 dots, second active).

Page 3: NET (Connection Status)

  • Animated signal strength: 4 bars that pulse upward when connected.
  • IP address centred, WebSocket status with live/dead dot indicator.
  • Latency readout if connected, "OFFLINE" in red if disconnected.
  • Signal bars animate with a subtle rising effect (1px per frame, cycling).

Live Overlay

  • 32px tall dark panel with slight transparency, slides up from bottom edge.
  • Shows parameter name and new value in large centred text.
  • Background: semi-transparent dark (#000000CC) to maintain page context.
  • Auto-dismisses after 2 seconds with reverse slide-down animation.

M5GFX Implementation Notes

  • Page slide: render both pages to offscreen sprites, pushSprite() with X-offset. At 30fps, 6 frames = 200ms transition. Each frame moves X by ~21px.
  • Overlay: single sprite, pushSprite(0, 128-overlayY) with overlayY animating from 0 to 32 over 4 frames.
  • Disconnect flash: M5.Lcd.drawRect(0, 0, 128, 128, RED) alternating with BLACK for 6 frames (3 cycles).
  • Total RAM: two 128x128x2 page sprites (32KB) + one 128x32x2 overlay sprite (8KB) = 40KB. Well within AtomS3 PSRAM.
