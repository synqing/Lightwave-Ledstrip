Design Rationale — Compact Pro

Philosophy

Combines the best of three paradigms: carousel navigation for information density, live parameter overlay for immediate feedback during encoder adjustments, and an always-visible persistent header for critical system state. The operator never loses awareness of connection status or zone system state, regardless of which page or overlay is active.

Persistent Header (16px — Never Changes)

The top 16 pixels are sacred — they remain constant across all pages and overlays. This is the "instrument panel" strip that an operator glances at without thinking.

  • Left: WiFi dot (4px, green / red) + WebSocket dot (4px, green / yellow / red). Two-dot status covers all connection failure modes at a glance.
  • Centre: Z:3 — zone count. Confirms the current zone topology without navigating to a settings page.
  • Right: ON in green or OFF in dark red — zone system power state. The most critical operational binary (are zones active?) is always visible.

Page Dots

Three dots at y=18, between header and content. The active page dot is #FFB000 amber, inactive dots are #333. Provides spatial orientation within the carousel without consuming significant vertical space (3px dots + 1px gaps = minimal).

Page 1: Zone Brightness (Amber Accent)

  • Three zone rows at 22px each: zone label, horizontal brightness bar (proportional fill), numeric percentage value. Each zone uses its assigned colour (Z1 red, Z2 green, Z3 blue).
  • Master row below zones: white bar with M label, full-width authority. The master controls overall output — visual weight reflects hierarchical importance.
  • Amber accent line (2px wide) on the left edge identifies this as the "zones" page even in peripheral vision.

Page 2: Effect Parameters (Violet Accent)

  • Effect name in amber at the top of the content area (12px font). Truncated with ellipsis for long names.
  • Speed parameter: blue bar with percentage. Speed is the most-adjusted effect parameter during performance.
  • EdgeMixer mode display: MODE: EDGE and SPREAD: 50%. These are reference-only values — EdgeMixer is configured via the K1 API, shown here for awareness.
  • Violet accent line distinguishes this page from zones. Violet reads as "creative/effects" in the DJ-mixer mental model.

Page 3: Connection Detail (Green Accent)

  • Full connection diagnostics: IP address, RSSI signal strength bars, WebSocket reconnect count.
  • RSSI rendered as a 5-segment bar (like a phone signal indicator). Each segment represents ~20dBm range. This is the "deep dive" for troubleshooting — the header dots tell you IF connected, this page tells you HOW WELL.
  • Reconnect count (RC:0) surfaces WebSocket instability before it becomes a visible problem. If this number is climbing, something is wrong even if the WS dot is green.
  • Green accent line — "network/health" semantic.

Live Overlay (Header Persists)

When an encoder is turned, the overlay appears over the carousel content but the header remains visible. This is the critical innovation: during a parameter change, the operator sees both the new value (large, central, unmissable) and the connection state (header, peripheral vision).

  • Parameter name in 10px at the top of the overlay area — identifies what is being changed.
  • Value rendered at 40px height, centred. Slightly smaller than the typical 48px to fit within the 108px content area while leaving room for the name and position bar. 40px is still legible from 2+ metres in a performance environment.
  • Position bar at the bottom of the overlay — horizontal fill bar showing the parameter's position within its full range. Provides spatial context that the number alone cannot.
  • Timeout: overlay disappears 1.5 seconds after the last encoder movement, returning to the current carousel page. The timeout is generous enough to read but short enough to not obscure the page during a show.

Colour System

  • #FFB000 — Amber. Active accent, effect names, active page dot. High visibility without alarm connotation.
  • #FF4444 — Zone 1 (red). Strongest visual weight for the primary zone.
  • #44FF44 — Zone 2 (green). Maximum contrast against Zone 1.
  • #4488FF — Zone 3 / Speed (blue). Cool complement, reused for speed to reduce colour count.
  • #9944FF — Violet. Effects/creative page accent. Not used for data, only for page identity.
  • #22CC44 — System green. Connection healthy, zones ON.
  • #CC2222 — System red. Connection failed, zones OFF.
  • #CCAA00 — System yellow. WebSocket reconnecting (not failed, not healthy).

Layout Budget

  • Header: y=0–15 (16px) — persistent, never scrolls or changes.
  • Page dots: y=17–19 (3px) — carousel position indicator.
  • Content area: y=22–127 (106px) — carousel pages or live overlay.
  • Accent line: x=0–1 (2px wide), y=22–127 — page identity colour on left edge.
  • Total: 16 + 3 + 2 (gap) + 106 = 127px (1px bottom margin for visual breathing room).

Accessibility on Hardware

All active text exceeds 7:1 contrast against #0a0a0a background. Zone colours form an RGB triad distinguishable with common colour vision deficiencies. Numeric values alongside all bars provide redundant encoding. Page accent colours use position (left edge) as a secondary cue alongside colour. The persistent header means connection failure is visible within 0.5 seconds of any glance, regardless of which page the operator was viewing.

Why "Compact Pro" Wins

Previous designs forced a choice: show everything (cluttered) or show one thing (blind to the rest). Compact Pro resolves this by separating critical (header — always visible) from contextual (carousel — swipeable) from transient (overlay — auto-dismiss). The 16px header investment pays for itself by eliminating the need to navigate back to check connection state. The overlay preserves the header because losing connection awareness during a parameter change is the exact moment you most need it.
