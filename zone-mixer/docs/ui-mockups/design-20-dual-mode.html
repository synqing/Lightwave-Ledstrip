Design Rationale

Dual-Mode Philosophy

A DJ mixer controller serves two fundamentally different use cases: live performance (where the operator needs instant visual feedback from a distance) and technical setup (where dense diagnostic data helps debug I2C buses, connection drops, or parameter tuning). Rather than compromise both into one mediocre layout, Design 20 gives each its own optimised display mode, toggled by a deliberate long-press to prevent accidental switching during performance.

Mode A: Operator

  • A1 — Zone Faders: Bold vertical bars fill most of the screen. Zone colours (#FF4444, #44FF44, #4488FF) are saturated for instant zone identification. The master bar is white and wider, visually anchoring the fader bank. Numeric values are rendered below each bar in 5x7 pixel font. Readable from 1m distance under stage lighting.
  • A2 — Effect + Speed: Large centred effect name (up to 10 chars) with speed as a horizontal bar beneath. The effect name dominates the screen — this is the most-glanced data point during live mixing. Connection dots in the header provide ambient status awareness.
  • Overlay: When any encoder changes a value, a 48px numeric overlay appears centre-screen for 2 seconds, then fades. This provides immediate haptic-to-visual feedback without leaving the current page. The overlay uses a semi-transparent dark background to remain readable over any page content.

Mode B: Tech

  • B1 — All Parameters: Dense 8-row text list showing every parameter value. Brightness per zone, speed, effect ID, zone enable states, EdgeMixer state — everything on one screen. Uses dim #888 labels with bright #FFF values for scanability. No graphics, maximum information density.
  • B2 — Connection Diagnostics: IP address, RSSI dBm, WebSocket state, reconnect count, poll cycle time, I2C error count. All numeric with units. This is the screen you stare at when troubleshooting why the controller is not responding.
  • B3 — Input Monitor: Scrolling event log showing the last 5 input events with timestamps. Format: E3 +1 (encoder 3, clockwise), S2 BTN (scroll unit 2 button press). Every input is logged here — no overlay in tech mode, because the log IS the feedback mechanism.

Mode Indicator

A small mode letter (A or B) is always rendered in the top-right corner at (122, 2). In Mode A it is amber #FFB000; in Mode B it is cyan #33CCFF. This is subtle enough not to distract during performance but always visible for orientation.

Colour System

  • #FFB000 — Amber accent, Mode A indicator. High visibility, "active" semantic.
  • #33CCFF — Cyan accent, Mode B indicator. Cool tone signals "diagnostic/technical".
  • #FF4444 — Zone 1 (warm red).
  • #44FF44 — Zone 2 (green).
  • #4488FF — Zone 3 (blue).
  • #FFFFFF — Master fader. White signals authority over all zones.
  • #0a0a0a — Background. Near-black with slight warmth to reduce eye fatigue.

Interaction Model

  • Long press (>800ms): Toggle between Mode A and B. The deliberate gesture prevents accidental mode switches during live performance. A brief flash animation confirms the switch.
  • Short press (<400ms): Cycle to next page within current mode. Mode A: A1 → A2 → A1. Mode B: B1 → B2 → B3 → B1.
  • No overlay in Mode B: Tech mode logs every input event to the B3 scrolling list instead. This gives persistent history rather than a transient 2-second popup — more useful for debugging input issues.

Implementation Notes

  • Mode state is a single boolean (isOperatorMode) persisted in NVS so it survives power cycles. Operators boot into their preferred mode.
  • All rendering uses the existing 5x7 pixel font. Mode A uses 2x scaled characters for the overlay (10x14 effective). No additional font assets required.
  • The overlay timer uses millis() comparison, not a FreeRTOS timer — simpler and sufficient for a 2-second timeout at 30+ FPS display refresh.
  • B3 input log is a 5-entry circular buffer of structs: {timestamp, source, value}. Memory cost: ~60 bytes. No heap allocation on input events.

Accessibility

Zone colours are chosen for deuteranopia safety: red/blue are distinguishable, and green has high luminance contrast against both. All text meets minimum 4.5:1 contrast ratio against #0a0a0a background. Mode B text uses #888 labels (5.3:1) and #FFF values (19.6:1).
